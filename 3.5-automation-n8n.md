# 외부 자동화: n8n으로 Claude Code를 넘어서

## Claude Code의 한계, n8n의 역할

Claude Code와 n8n은 PM의 워크플로우 자동화 스택에서 서로 다른 역할을 담당합니다.

- **Claude Code** = "내가 말해야 움직인다" (수동 트리거)
  - PM이 `/today`, `/status`, `/prd` 명령어를 실행하면 작동
  - 실시간 분석과 깊은 사고가 필요한 작업에 최적
  - 하지만 PM이 매번 명령어를 입력해야 함

- **n8n** = "내가 자고 있어도 움직인다" (스케줄/이벤트 기반)
  - 정해진 시간에 자동 실행 (매일 7AM 뉴스봇 등)
  - 특정 이벤트 발생 시 자동 반응 (이메일 도착, 폼 제출 등)
  - PM의 개입 없이 24/7 작동

- **함께 쓰면**: Claude Code(실행형) + n8n(자동화형) = **PM의 24시간 비서**
  - 반복적인 데이터 수집은 n8n이 자동으로
  - 판단과 분석은 Claude Code가 온디맨드로
  - PM은 전략과 의사결정에만 집중

## n8n이란

n8n은 오픈소스 워크플로우 자동화 플랫폼입니다. PM이 알아야 할 핵심 특징:

- **400+ 앱 연동**: Gmail, Slack, Notion, Google Sheets, RSS, Linear, Airtable, HubSpot 등
- **코드 없는 비주얼 빌더**: 드래그&드롭으로 워크플로우 구성
- **셀프호스팅 가능**: 회사 서버에 설치하면 데이터 완전 통제 (프라이버시 보호)
- **무료 오픈소스** vs **클라우드 버전** (비용 트레이드오프)
- **다양한 트리거**: 스케줄, 웹훅, 이벤트, 데이터베이스 변경 감지 등

n8n은 "자동화를 민주화"한다는 철학을 가지고 있어서, PM도 커스텀 워크플로우를 쉽게 만들 수 있습니다.

## PM을 위한 4가지 자동화 워크플로우

### 워크플로우 1: 자동 뉴스 다이제스트

**목적**: PM이 매일 아침 AI/LLM/에이전트 관련 뉴스를 자동으로 받도록

```
n8n (매일 아침 7AM Schedule Trigger):
  ├─ RSS Feed 10개 소스 수집
  │  └─ Product Hunt, HN, Medium, 업계 뉴스레터 등
  ├─ AI 노드로 각 뉴스 처리
  │  ├─ 요약 생성 (핵심 내용 2줄)
  │  └─ 관련도 스코어링 (AI/LLM/자동화 관련도 0~1)
  ├─ 필터링
  │  └─ 관련도 > 0.6만 통과
  ├─ Notion DB "뉴스 다이제스트" 테이블에 저장
  │  └─ [제목] [출처] [요약] [스코어] [날짜] 필드
  └─ Slack #news 채널에 하이라이트 발송
     └─ "📰 오늘의 뉴스 5건 - 모두 Notion에서 확인"

Claude Code /today 실행 시:
  └─ Notion MCP로 오늘 뉴스 다이제스트 조회
     └─ PM 브리핑에 "오늘의 AI 뉴스" 섹션으로 자동 포함
```

**효과**: PM이 뉴스 스캔 시간 30분/일 절약, 놓치는 정보 제로

---

### 워크플로우 2: 사용자 피드백 수집 & 분석

**목적**: 고객 피드백을 자동으로 수집, 분류, 분석하여 인사이트 제공

```
n8n (Form/Webhook 트리거 - 피드백 폼 제출 시):
  ├─ 피드백 폼 데이터 수신
  ├─ Google Sheets "원본 피드백" 시트에 저장
  │  └─ 타임스탬프, 사용자, 내용, 카테고리 등
  ├─ AI 노드에서 처리
  │  ├─ 감성 분석 (긍정/중립/부정, -1.0 ~ +1.0)
  │  ├─ 주제 분류 (기능 요청/버그/UI/가격 등)
  │  └─ 감정 강도 스코어
  ├─ Notion DB "고객 피드백" 테이블에 구조화 저장
  │  └─ [날짜] [출처] [원문] [감성] [주제] [조치 상태] 필드
  └─ 조건부 실행
     └─ IF 부정적 피드백 (감성 스코어 < -0.3)
        └─ Slack #cs-urgent 채널에 즉시 알림
           └─ "⚠️ 부정적 피드백: [사용자] - [내용]"

Claude Code /status 실행 시:
  └─ Notion MCP로 이번 주 피드백 요약 조회
     └─ 주간 보고서에 "고객 목소리" 섹션으로 자동 포함
        └─ 주요 요청사항 TOP 5, 해결해야 할 버그 등
```

**효과**: 고객 목소리를 놓치지 않으면서 PM의 분류 작업 제거

---

### 워크플로우 3: 콘텐츠 발행 파이프라인

**목적**: Claude Code가 초안 작성 → PM 승인 → n8n이 자동 게시

```
Step 1: Claude Code 실행 (PM이 수동 트리거)
  └─ /linkedin "AI 에이전트의 미래"
     ├─ 콘텐츠 초안 생성
     └─ Notion "콘텐츠 캘린더" DB에 자동 저장
        └─ [제목] [초안] [상태: "작성중"] [예정일] 필드

Step 2: PM이 Notion에서 리뷰 & 승인
  └─ 초안 읽기 → 필요시 수정 → status를 "approved"로 변경

Step 3: n8n (Notion 트리거 - status 변경 감지 시)
  ├─ status = "approved" 감지
  ├─ Notion에서 콘텐츠 추출 (제목, 본문, 이미지)
  ├─ LinkedIn API로 자동 게시
  ├─ Slack #marketing 채널에 알림
  │  └─ "✅ [제목] 발행 완료! LinkedIn 링크: [URL]"
  └─ Notion의 status를 "published"로 자동 변경
     └─ 발행 시간과 URL도 저장
```

**효과**: 검수는 PM이 하고, 실제 발행은 자동화 → 일관성 + 효율성

---

### 워크플로우 4: 이메일 CS 자동 분류

**목적**: 들어오는 이메일을 AI가 자동으로 분류하고 초기 대응 자동화

```
n8n (Gmail 트리거 - 새 이메일 수신 시):
  ├─ 이메일 수신 (제목, 본문, 발신자)
  ├─ AI 노드에서 분류
  │  └─ 카테고리: 제품 문의 / 기술 지원 / 결제 문제 / 일반 문의
  │  └─ 우선도: 높음(긴급) / 중간 / 낮음
  │
  ├─ IF 제품 문의
  │  ├─ Notion "리드" DB에 자동 등록
  │  │  └─ [발신자] [문의 내용] [분류] [날짜] [상태: "신규"]
  │  ├─ AI가 응답 초안 생성
  │  │  └─ "감사합니다. 우리 제품에 관심 주셔서..."
  │  └─ Slack #sales에 알림
  │     └─ "🔥 새로운 리드: [발신자] - [내용]"
  │
  ├─ IF 기술 지원
  │  ├─ Linear에 티켓 자동 생성
  │  │  └─ [제목] [상세 내용] [우선도] [담당자: 자동 배정]
  │  └─ Slack #support에 알림
  │
  └─ 모든 이메일 내용을 Google Sheets 로그에 저장
     └─ 나중에 PM이 트렌드 분석 가능
```

**효과**: CS 팀이 중복 분류 시간 제거, 놓치는 고객 문의 감소

## 3중 레이어 아키텍처

PM의 일상적 워크플로우는 3개 도구가 계층적으로 작동합니다:

```
┌──────────────┐  ┌──────────────┐  ┌────────────────────┐
│ Claude.ai    │  │ Claude Code  │  │       n8n          │
│ (대화형)      │  │ (실행형)      │  │   (자동화형)        │
├──────────────┤  ├──────────────┤  ├────────────────────┤
│ • 전략 논의   │  │ • /today     │  │ • 스케줄 트리거    │
│ • PRD 작성   │  │   (오늘 브리징)│ │   (매일 7AM 등)   │
│ • 문서 리뷰   │  │ • /prd       │  │ • 웹훅 트리거      │
│ • 즉석 분석   │  │   (문서 작성) │  │   (이벤트 기반)   │
│ • 전체 맥락   │  │ • /status    │  │ • 뉴스봇 실행     │
│   파악       │  │   (주간 보고) │  │ • 피드백 수집     │
│              │  │ • /competitor │  │ • 이메일 분류     │
│              │  │   (경쟁 분석) │  │ • 자동 게시       │
└──────┬───────┘  └──────┬───────┘  └────────┬──────────┘
       │ (고수준 질문)    │ (실행 명령어)      │ (자동 실행)
       └────────┬─────────┴─────────────────┘
                │
     공유 데이터 레이어 (모든 도구가 접근)
     ┌────────────────────────────────────────┐
     │ [Notion] ←→ [Linear] ←→ [Slack]        │
     │ ←→ [Gmail] ←→ [Sheets] ←→ [RSS]       │
     └────────────────────────────────────────┘
```

**데이터 흐름 예시**:
1. Claude.ai에서 "주간 인사이트 리포트" 전략 논의
2. Claude Code가 `/status` 실행 → Notion에서 주간 데이터 수집
3. n8n이 백그라운드에서 매일 뉴스/피드백/이메일 자동 수집 → Notion 저장
4. 모든 데이터가 Notion에 통합되어 Claude Code가 분석에 활용

## Claude Code vs Cursor+MCP vs n8n: 언제 무엇을 쓰는가

| 상황 | 도구 | 이유 |
| --- | --- | --- |
| PRD 작성 (5000자 이상) | Claude Code | 긴 문서, 반복 수정, 로컬 저장 |
| 매일 뉴스 자동 수집 | n8n | 스케줄 자동화, PM 개입 불필요 |
| 경쟁사 분석 (웹 검색) | Claude Code | 웹 검색 + 깊은 분석 능력 |
| 고객 피드백 분류 | n8n | 이벤트 기반 자동화, 실시간 처리 |
| 전략 논의 (초안 단계) | Claude.ai | 대화형, 반복 Q&A, 즉석 아이디어 |
| 주간 보고서 작성 | Claude Code | 최신 데이터 수집 + 포매팅 |
| 소셜 미디어 게시 | n8n | 정해진 시간 자동 발행 |
| 긴급 경쟁 뉴스 분석 | Claude.ai | 실시간 대화, 빠른 판단 |

## PM 역할 확장 포인트

### Before (자동화 전)
```
PM의 일일 반복 작업:
  ┌─ 7:30 - 뉴스레터 읽기 (30분)
  ├─ 9:00 - 이메일 분류 (20분)
  ├─ 10:00 - 고객 피드백 정리 (25분)
  ├─ 2:00 - 주간 보고서 데이터 수집 (40분)
  └─ Total: 약 2시간/일 반복 작업
     → PM이 전략과 판단에 집중할 시간 부족
```

### After (n8n + Claude Code)
```
n8n이 자동으로 처리:
  ├─ 7:00 - 뉴스 수집 & 요약 ✓ (자동)
  ├─ 8:30 - 이메일 분류 & 분류 ✓ (자동)
  ├─ 10:00 - 고객 피드백 수집 & 분석 ✓ (자동)
  └─ → Notion에 모두 저장됨

PM의 역할 변화:
  ├─ Claude Code /today (5분)
     └─ 오늘의 요약 + 뉴스 + 피드백 + 이슈 한눈에
  ├─ 전략 의사결정 (2시간)
     └─ "어떤 기능을 우선순위로 할까?" 등
  └─ Claude.ai 대화 (30분)
     └─ 새로운 아이디어 브레인스토밍
```

**결과**: PM이 진정한 "전략가"로 역할 강화 → 가치 있는 판단에만 시간 투자

## ⚠️ 주의: 자동화의 함정

### 함정 1: 알림 과다 → 알림 피로 → 모두 무시

```
❌ 나쁜 예: 모든 이벤트를 Slack에 알림
  └─ "새 이메일", "피드백 1개", "뉴스 1건" 등
  └─ Slack 알림 폭주 → PM이 무시하게 됨

✅ 좋은 예: 필터링 후 중요한 것만 알림
  └─ n8n의 조건부 노드 사용
  └─ IF (감성 < -0.3) OR (우선도 = 높음) THEN Slack 알림
  └─ → 하루 5개 알림만 → PM이 모두 읽음
```

### 함정 2: 자동 게시 → 검수 없이 발행 → 품질 저하

```
❌ 위험: 자동화만으로 콘텐츠 발행
  └─ AI 요약 오류 → 잘못된 정보 발행 → 신뢰도 손상

✅ 안전: 자동화 + 수동 승인 조합
  └─ n8n이 초안 작성 → Notion에 저장
  └─ PM이 검수 후 "approved" → n8n이 발행
  └─ → 자동화의 효율 + 검수의 품질 보장
```

### 함정 3: 과도한 자동화 → PM의 맥락 파악력 저하

```
❌ 위험: 모든 것을 자동화
  └─ PM이 기초 데이터를 직접 안 봄 → 고객 이해도 감소
  └─ "왜 이 지표가 떨어졌나?" 물었을 때 대답 못함

✅ 균형: 중요한 부분은 수동으로 유지
  └─ n8n: 반복적인 데이터 수집만
  └─ Claude Code: PM이 주 1회 직접 실행 (맥락 유지)
  └─ → 효율성 + 이해도 둘 다 확보
```

## 실습 과제

### Level 2: n8n에서 RSS → AI 요약 → Notion DB 저장 워크플로우

**목표**: 기술 뉴스 RSS 3개 소스를 n8n으로 자동 수집하여 Notion에 저장

**구체적 단계**:
1. n8n 회원가입 (또는 셀프호스팅 설치)
2. Notion API 통합 설정 (Notion Internal Integration Token)
3. "뉴스 다이제스트" 데이터베이스 생성 (필드: 제목, 출처, 요약, URL, 날짜)
4. n8n 워크플로우 구성:
  - Schedule Trigger: 매일 7AM
  - 3개 RSS Feed 노드 (예: TechCrunch)
  - AI 노드: 각 뉴스 요약 (2줄)
  - Notion Create DB Entry: 요약본을 Notion에 저장
5. 테스트 실행 후 매일 아침 데이터가 Notion에 쌓이는지 확인

**제출 내용**: n8n 워크플로우 스크린샷 + Notion 데이터 샘플

---

### Level 3: Claude Code /today와 n8n 뉴스봇을 연결하는 전체 파이프라인

**목표**: PM이 Claude Code `/today` 실행하면 → n8n에서 수집한 뉴스가 자동으로 포함되는 완전한 브리핑 시스템

**구체적 단계**:
1. Level 2 워크플로우 완성 (n8n RSS → Notion 저장)
2. n8n 추가 설정:
  - Notion 트리거: 매일 7AM 후 상태 업데이트 (아직 미읽)
  - Slack 알림: "오늘의 뉴스 5건 준비됨" 메시지 발송
3. Claude Code MCP 설정 (Notion MCP 활성화):
  - Notion 데이터베이스 쿼리 권한 설정
  - 필터: today 날짜 뉴스만 조회
4. `/today` 커맨드 수정:
  - "## 📰 오늘의 AI 뉴스" 섹션 추가
  - Notion MCP로 뉴스 다이제스트 조회 & 자동 포함
  - 형식: "[제목] - [요약] [출처]"
5. 전체 테스트:
  - n8n이 뉴스 자동 수집 (새벽 7AM)
  - PM이 아침 9AM에 `/today` 실행
  - 브리핑에 자동으로 최신 뉴스가 포함됨

**제출 내용**:
- n8n 전체 워크플로우 (RSS + Notion + Slack)
- Claude Code `/today` MCP 설정 코드
- 실제 동작하는 브리핑 결과 스크린샷

## 마무리: PM의 자동화 철학

자동화는 "손으로 하는 반복 작업"을 없애기 위한 것이 아니라, **"PM의 판단 시간을 늘리기 위한" 도구**입니다.

- **n8n**: 데이터 수집을 자동화 (24/7 작동)
- **Claude Code**: 수집한 데이터로 분석과 초안 작성 (온디맨드)
- **Claude.ai**: 분석 결과로 전략 논의 (대화형)

이 3계층이 잘 작동할 때, PM은:
- 반복 작업에서 해방되고
- 데이터 기반 의사결정을 할 수 있고
- 더 많은 고객 이해를 통해 더 나은 제품을 만들 수 있습니다.
