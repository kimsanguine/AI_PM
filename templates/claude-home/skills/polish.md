# /polish — 글 다듬기 스킬

## Task
주어진 글의 **의미는 유지**한 채 톤·문장 구조·단어를 다듬는다.

## Scope
- 한국어 prose 만 대상. 코드 블록·표·링크는 건드리지 않는다.
- 단락 추가/삭제 금지. 단락 내부에서만 수정.
- 사실 변경 금지. 숫자·고유명사·인용은 그대로.

## Length
- 입력 글이 줄어들 수는 있으나, **30% 이상 줄어들면 멈추고 사용자에게 확인**.

## Format
출력은 두 블록:

```
<polished>
... 다듬은 글 전문 ...
</polished>

<diff_summary>
- L12: "다양한 솔루션" → "이 5가지 방법"
- L34: 수동태 → 능동태
- 전체: 무의미 형용사 8개 제거
</diff_summary>
```

## 룰

- 첫 문장을 결론으로 끌어올린다 (역피라미드).
- 한 절 = 한 주장. 두 주장이 섞이면 분리.
- 회사문서 톤 단어("솔루션", "관련하여") 제거.
- 1개짜리 불릿은 산문으로.
- 표는 4행 3열 미만이면 산문으로.

## Examples

<example>
<input>다양한 솔루션을 활용하여 효과적인 사용자 경험을 제공할 수 있습니다.</input>
<output>로그인 시간을 4단계에서 2단계로 줄였다.</output>
</example>

<example>
<input>이 기능과 관련하여 여러가지 개선점들이 발견되었습니다.</input>
<output>이 기능에서 3가지를 개선했다: 속도, 에러 메시지, 모바일 레이아웃.</output>
</example>
