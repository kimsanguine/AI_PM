## 리뷰의 4단계 (검증 사다리)

순서대로, 위에서 통과 못 하면 아래는 의미 없다.

1. **자동 포맷** — markdownlint / textlint 통과 여부.
2. **링크·테스트** — 깨진 링크 0, 명령 예시 동작 검증.
3. **시각·렌더** — GitHub 렌더링/스크린샷 확인.
4. **LLM 리뷰** — 별도 세션에서 본 가이드의 룰로 재검토.

## 무엇을 찾는가 (coverage 우선)

찾을 때는 **coverage 우선**, 필터는 나중에.

- 사실 오류 (수치, 모델 ID, 링크)
- 모순 (앞 절 vs 뒷 절)
- 불완전한 주장 (근거 없는 단정)
- 일반화 오류 ("apply to every" 인데 한 곳만 적용)
- 톤 일관성 (직설/한국어/이모지 룰)
- 4섹션 누락 (슬래시·스킬의 Task/Scope/Length/Format)

## 리포팅 포맷

```
## 발견
- L42: <인용 한 줄> — 문제: <한 줄> — 심각도: high/mid/low/nit
- L88: ...

## 제안 패치
- L42: "X" → "Y"
- ...

## 통과 항목
- 검증 사다리 1단·2단 통과
- 모델 ID 일치
```

## 안티패턴

- "전반적으로 좋아 보입니다" 같은 무내용 리뷰 금지.
- 심각도 표기 없이 발견만 나열 금지.
- 동의(rubber-stamp)만 하는 리뷰는 리뷰가 아니다.
