## 작업 규칙

### 문서 작성 규칙
- 모든 PRD는 `prd/` 폴더에 저장
- 파일명 규칙: `[project-name]-[doc-type]-v[version].md`
- YAML front matter 필수 (type, due, tags, status, priority)
- 한국어로 작성, 기술 용어는 영문 병기

### 분석 규칙
- 데이터 분석 결과에는 반드시 [근거] 태그 포함
- 신뢰도 표시: 🟢 높음(70%+) / 🟡 중간(40-70%) / 🔴 낮음(40%-)
- 단일 소스 기반 결론에는 [단일 소스 주의] 표시
- 상관관계와 인과관계를 명확히 구분

### Human-in-the-Loop 규칙
- 외부 공유 문서는 반드시 PM 확인 후 저장
- KPI 수치 변경 시 PM 확인 필수
- 경쟁사 분석의 주관적 해석은 PM 확인 필수
- 파일 삭제, Git push, 외부 API 호출 전 PM 확인 필수
- 해석이 2가지 이상 가능한 경우 옵션을 제시하고 PM에게 선택 요청

### 자동 실행 허용 영역
- 파일 포맷 변환 (CSV → MD, JSON → 테이블)
- 정해진 템플릿 기반 문서 생성
- 데이터 시각화 (차트, 그래프, Mermaid 다이어그램)
- 코드 포맷팅 및 린팅
