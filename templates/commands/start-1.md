# /start-1 — 입문 트랙 (Part1~2)

## Task

Claude Code 를 처음 쓰는 PM 이 "설치 → 도구 지도 이해 → CLAUDE.md 로 맥락 주기" 까지
직접 손으로 해보며 익히도록, 한 번에 한 단계씩 끌고 간다.

## Scope

- 학습자 대신 과제를 끝내지 않는다. 학습자가 직접 입력하고 실행하게 한다.
- 한 단계가 끝날 때마다 30초 검증 질문을 던지고, 답을 받은 뒤에만 다음 단계로 넘어간다.
- 다루는 범위는 Part1~2 입문까지. 스킬·MCP 는 `/start-2` 로 넘긴다.

## Length

- 각 단계 안내는 400자 이내. 한 응답에 한 단계만 제시한다.

## Format

각 단계는 [읽을 챕터] + [직접 해볼 과제] + [30초 검증] 세 블록으로 구성한다.
한 단계를 안내한 뒤 반드시 멈추고 학습자 응답을 기다린다.

### 1단계 — 설치하고 첫 대화 열기

- 읽을 챕터: [설치부터 첫 대화까지](../../docs/part1-foundations/1.3-install-and-first-run.md)
- 직접 해볼 과제: 터미널에서 Claude Code 를 실행하고, "이 폴더에 어떤 파일이 있는지 한 줄로 설명해줘" 라고 한 줄 요청을 직접 입력해 본다.
- 30초 검증: 방금 받은 답이 실제 폴더 내용과 맞나요? (예/아니오 + 한 줄 소감)

### 2단계 — 도구 지도 이해하기

- 읽을 챕터: [Claude Code: 왜 CLI인가](../../docs/part1-foundations/1.2-what-is-claude-code.md), 이어서 [파일 다루기: @참조와 입력 방식](../../docs/part2-basics/2.1-files-and-input.md)
- 직접 해볼 과제: 아무 파일이나 `@파일명` 으로 참조해 "이 파일 핵심을 3줄로 요약해줘" 라고 요청해 본다.
- 30초 검증: `@` 로 파일을 참조하는 것과 그냥 말로 설명하는 것의 차이를 한 문장으로 말해 보세요.

### 3단계 — 분석 깊이 조절하기

- 읽을 챕터: [모드와 분석 깊이: 상황별 최적 설정](../../docs/part2-basics/2.2-modes-and-depth.md)
- 직접 해볼 과제: 같은 질문을 가볍게 한 번, 깊게 한 번 요청해 답의 길이·근거 차이를 비교해 본다.
- 30초 검증: 언제 깊은 분석이 필요하고 언제 과한지, 본인 업무 예시 1개로 답해 보세요.

### 4단계 — CLAUDE.md 로 맥락 주기

- 읽을 챕터: [CLAUDE.md: 프로젝트의 두뇌를 만들다](../../docs/part2-basics/2.3-project-memory.md)
- 직접 해볼 과제: 현재 폴더에 `CLAUDE.md` 를 만들고, "이 프로젝트는 무엇이고 누구를 위한 것인지" 3줄을 직접 적어 본다. 그다음 같은 질문을 다시 던져 답이 달라지는지 본다.
- 30초 검증: CLAUDE.md 작성 전후로 답이 어떻게 달라졌나요? 한 문장으로.

### 마무리

4단계를 마치면, 다음 트랙으로 스킬·슬래시 커맨드·MCP 를 다루는 `/start-2` 를 안내하고 멈춘다.
