# /start-2 — 활용 트랙 (Part2~3)

## Task

기본 대화에 익숙한 PM 이 "서브에이전트 → 슬래시 커맨드 → 커스텀 스킬 → MCP 연동" 으로
반복 업무를 자동화하는 감각을 직접 만들어 보도록, 한 번에 한 단계씩 끌고 간다.

## Scope

- 학습자 대신 과제를 끝내지 않는다. 직접 만들고 실행하게 한다.
- 한 단계가 끝날 때마다 30초 검증 질문을 던지고, 답을 받은 뒤에만 다음 단계로 넘어간다.
- 다루는 범위는 Part2~3 활용까지. 전체 워크플로 관통은 `/start-3` 로 넘긴다.

## Length

- 각 단계 안내는 400자 이내. 한 응답에 한 단계만 제시한다.

## Format

각 단계는 [읽을 챕터] + [직접 해볼 과제] + [30초 검증] 세 블록으로 구성한다.
한 단계를 안내한 뒤 반드시 멈추고 학습자 응답을 기다린다.

### 1단계 — 서브에이전트로 리뷰 받기

- 읽을 챕터: [커스텀 서브에이전트: AI 리뷰어 팀 만들기](../../docs/part2-basics/2.4-custom-subagents.md)
- 직접 해볼 과제: 짧은 문서나 메시지를 하나 골라 "비판적 리뷰어 관점으로 약점 3가지만 짚어줘" 라고 직접 요청해 본다.
- 30초 검증: 일반 답변과 리뷰어 관점 답변의 차이를 한 문장으로 말해 보세요.

### 2단계 — 슬래시 커맨드 이해하기

- 읽을 챕터: [슬래시 커맨드: 반복 워크플로우를 원클릭으로](../../docs/part3-advanced/3.3-slash-commands.md)
- 직접 해볼 과제: 본인이 매주 반복하는 PM 업무 1개를 떠올리고, "이걸 슬래시 커맨드로 만든다면 어떤 단계로 나눌까" 를 직접 적어 본다.
- 30초 검증: 그 업무를 단계로 쪼갰을 때 가장 자동화 가치가 큰 단계는 무엇인가요?

### 3단계 — 커스텀 스킬 만들기

- 읽을 챕터: [커스텀 스킬: 자주 쓰는 워크플로우를 저장하고 재사용하기](../../docs/part3-advanced/3.4-custom-skills.md)
- 직접 해볼 과제: 2단계에서 떠올린 업무를 "Task / Scope / Length / Format" 4섹션으로 한 장 초안을 직접 적어 본다. (이 코스의 커맨드 파일이 그대로 예시다.)
- 30초 검증: 4섹션 중 Scope 에 "하지 않을 것" 을 적는 게 왜 중요할까요? 한 문장으로.

### 4단계 — MCP 로 도구 연결하기

- 읽을 챕터: [MCP 연동: PM의 도구를 하나로 연결하기](../../docs/part3-advanced/3.1-mcp-integration.md)
- 직접 해볼 과제: 본인이 매일 쓰는 도구(이슈 트래커·문서·메신저 등) 중 Claude 와 연결하면 가장 손이 덜 갈 것 하나를 고르고, 그 이유를 직접 적어 본다.
- 30초 검증: 그 도구를 연결하면 어떤 반복 작업이 사라지나요? 한 문장으로.

### 마무리

4단계를 마치면, Discovery → Definition → Delivery 전체 흐름을 한 번에 관통하는 `/start-3` 을 안내하고 멈춘다.
