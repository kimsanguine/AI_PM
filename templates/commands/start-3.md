# /start-3 — 워크플로 관통 트랙 (Part4~6)

## Task

PM 의 핵심 흐름인 Discovery → Definition → Delivery 를 작은 가상 과제 하나로 끝까지 관통한다.
각 단계에서 실제 챕터를 읽고, 직접 산출물 한 조각을 만들어 다음 단계로 넘긴다.

## Scope

- 학습자 대신 산출물을 완성해 주지 않는다. 각 단계의 한 조각을 직접 만들게 한다.
- 한 단계가 끝날 때마다 30초 검증 질문을 던지고, 답을 받은 뒤에만 다음 단계로 넘어간다.
- 하나의 가상 과제를 1회 관통하는 것이 목표다. 완성도보다 흐름 체득이 우선.

## Length

- 각 단계 안내는 450자 이내. 한 응답에 한 단계만 제시한다.

## Format

먼저 학습자에게 작은 가상 과제 하나를 정하게 한다. (예: "신규 사용자 온보딩 개선")
그다음 단계마다 [읽을 챕터] + [직접 해볼 과제] + [30초 검증] 으로 진행하고, 매 단계 후 멈춘다.

### 1단계 — Discovery: 신호 모으기

- 읽을 챕터: [유저 리서치 데이터 분석](../../docs/part4-discovery/4.1-discovery-user-research.md), 필요하면 [경쟁사 분석: 구조화된 인텔리전스](../../docs/part4-discovery/4.2-discovery-competitive-analysis.md)
- 직접 해볼 과제: 정한 과제와 관련해 떠오르는 사용자 불편 3가지를 직접 적고, 각각이 "관찰된 사실" 인지 "추측" 인지 라벨을 붙여 본다.
- 30초 검증: 3개 중 실제 근거가 있는 것은 몇 개인가요? 추측은 어떻게 검증할 수 있나요?

### 2단계 — Definition: PRD 한 조각 쓰기

- 읽을 챕터: [PRD 작성: AI와의 소크라틱 대화](../../docs/part5-definition/5.1-definition-write-prd.md)
- 직접 해볼 과제: 1단계 신호 중 가장 강한 것 하나를 골라, "문제 정의 1문장 + 성공 지표(OMTM) 1개 + 범위에서 뺄 것 1개" 만 직접 적어 본다.
- 30초 검증: 성공 지표가 "측정 가능한가요"? 측정 방법을 한 문장으로 말해 보세요.

### 3단계 — Definition: 전략 한 줄 세우기

- 읽을 챕터: [전략 문서 작성: Rumelt의 전략 커널](../../docs/part5-definition/5.2-definition-product-strategy.md)
- 직접 해볼 과제: "왜 지금 이 문제를 푸는가" 를 한 문장으로 직접 적는다. (진단 → 방침 → 행동 중 진단 한 줄)
- 30초 검증: 그 한 줄이 "당연한 말" 인가요, 아니면 선택과 포기가 담겼나요?

### 4단계 — Delivery: 프로토타입 한 조각

- 읽을 챕터: [4.1 바이브 코딩: PM이 직접 프로토타입 만들기](../../docs/part6-delivery/6.1-delivery-vibe-coding.md), 시각 자료가 필요하면 [4.2 비주얼 에셋: 페르소나부터 마케팅 이미지까지](../../docs/part6-delivery/6.2-delivery-visual-assets.md)
- 직접 해볼 과제: 2단계 PRD 조각을 근거로, 화면 1개의 핵심 요소를 글로 묘사하고 "이 화면 하나를 간단한 프로토타입으로 만들어줘" 라고 직접 요청해 본다.
- 30초 검증: 나온 프로토타입이 2단계 성공 지표를 움직일 수 있나요? 한 문장으로.

### 마무리

4단계를 마치면 한 흐름을 관통한 것이다. 공개·배포가 궁금하면 [4.3 GitHub & 배포: 아이디어를 세상에 공개하기](../../docs/part6-delivery/6.3-delivery-github-deploy.md) 를 권하고, 전체 코스를 다시 보려면 `/start` 로 돌아가도록 안내한 뒤 멈춘다.
