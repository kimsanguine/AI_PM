# 파일 다루기: @참조와 입력 방식

## 개요
Claude Code의 핵심 강점은 실제 프로젝트 파일을 직접 분석하고 변환할 수 있다는 것입니다. PM이 엑셀 데이터, 마크다운 문서, CSV 파일을 @참조로 지정하면, Claude는 그 내용을 이해하고 즉시 분석, 변환, 개선 아이디어를 제시합니다. @참조 외에 프롬프트 창에 드래드앤드랍으로 파일을 옮기는 것도 가능합니다.

---

## 1. @파일 참조 기본 사용법

### 1.1 단일 파일 참조

**사용 문법:**
```
> @파일이름 [질문/요청]
```

**예시:**
```
> @quarterly-report.csv 이번 분기 매출 트렌드를 분석해줘

Claude Code가 반환:
📊 분석 결과:
- 1월: $245K (전월 대비 +12%)
- 2월: $267K (전월 대비 +9%)
- 3월: $289K (전월 대비 +8%)

주요 발견:
1. 지속적인 성장세 (평균 +9.7%)
2. 마케팅 캠페인 반응: 2월 이메일 캠페인 후 전환율 8% 상승
3. 위험 신호: 3월 말 성장률 둔화 트렌드
```

### 1.2 복수 파일 참조 (병렬 분석)

**사용 문법:**
```
> @file1.csv @file2.md @file3.json [교차 분석 요청]
```

**예시:**
```
> @funnel-data.csv @user-feedback.md 두 자료를 교차 분석해서 이탈 원인을 찾아줘

Claude Code가 반환:
🔍 교차 분석 결과:

이탈 단계별 원인:
1. 가입 화면 (32% 이탈)
   - 피드백: "너무 많은 필드를 입력하게 함"
   - 데이터 근거: 가입 단계 평균 소요시간 4분 43초 (벤치마크 2분)
   → 권장: 필수 필드 3개 → 2개로 축소

2. 가격 책정 화면 (18% 이탈)
   - 피드백: "경쟁사 대비 30% 비쌈"
   - 데이터 근거: 가격 확인 후 즉시 이탈률 18% (전체 평균 8%)
   → 권장: 월/연 구독 옵션 추가, 얼리버드 40% 할인 시범

3. 결제 화면 (9% 이탈)
   - 피드백: "카드 정보 입력 후 기다리는 시간이 느림"
   - 데이터 근거: 결제 평균 처리 시간 8초
   → 권장: 로딩 상태 명확 표시, Stripe 최적화
```

### 1.3 폴더 참조 (배치 처리)

**사용 문법:**
```
> @폴더명/ [배치 분석/종합 요청]
```

**예시:**
```
> @user-interviews/ 이 폴더의 인터뷰 노트를 종합해서 핵심 사용자 요구사항 Top 5를 뽑아줘

Claude Code가 반환:
📝 인터뷰 종합 분석 (5개 인터뷰, 총 47분)

Top 5 사용자 요구사항:
1. ⭐⭐⭐⭐⭐ 대시보드 커스터마이징 (5/5 언급)
   - 각 역할별로 보고 싶은 메트릭이 다름
   - Sarah(PM): 상태 + 마일스톤
   - Mike(엔지니어): 버그 수 + 배포 빈도

2. ⭐⭐⭐⭐ Slack 연동 (4/5 언급)
   - 알림을 Slack에서 받고 싶음
   - 대시보드를 Slack에 직접 포함하고 싶음

3. ⭐⭐⭐⭐ 벌크 작업 (4/5 언급)
   - 10개 이상 항목을 한 번에 수정하고 싶음
   - 반복되는 설정을 템플릿화하고 싶음

4. ⭐⭐⭐ API 접근 (3/5 언급)
   - 자체 보고 도구와 연동하고 싶음
   - 데이터 내보내기 자동화

5. ⭐⭐⭐ 모바일 앱 (3/5 언급)
   - 회의 중 빠른 조회 필요
   - 긴급 승인 처리
```

## 2. 이미지 입력: 스크린샷 분석

### 2.1 스크린샷 붙여넣기

**사용 문법:**
```
1. 스크린샷 캡처 (Mac: Cmd+Shift+4 / Windows: Win+Shift+S)
2. Claude Code에 Ctrl+V (또는 Cmd+V)
> [스크린샷이 자동 삽입됨]
  [질문/요청]
```

**예시:**
```
> [대시보드 스크린샷 붙여넣기]
  이 대시보드에서 이상 징후를 찾아줘. 특히 이번 분기와
  지난분기를 비교해서 뭔가 빨간 불이 켜진 지표가 있는지 봐줘

Claude Code가 반환:
🚨 이상 징후 3개 발견:

1. [심각] 신규 사용자 확보 -23% (week-over-week)
   - 지난주: 1,245명
   - 이번주: 958명
   - 원인 분석 필요: 마케팅 채널 점검, 유기적 트래픽 추이 검토

2. [경고] 평균 세션 지속시간 -8분 (지난분기 대비)
   - 지난분기: 18분 44초
   - 이번분기: 10분 32초
   - UI 변경? 새 기능 진출? 온보딩 문제?

3. [주목] 프리미엄 전환율 -4% (지난분기 대비)
   - 지난분기: 12%
   - 이번분기: 8%
   - 시점: 2주 전 가격인상 후 하락 시작
```

## 3. 지원 파일 포맷

Claude Code가 기본 지원하는 파일 타입:

| 포맷 | 활용 시나리오 |
| --- | --- |
| .md | 문서, PRD, 회의록, 제안서 |
| .csv | 데이터, 메트릭, 사용자 목록 |
| .json | API 응답, 설정 파일, 구조화된 데이터 |
| .pdf | 리포트, 계약서, 프레젠테이션 (텍스트 추출) |
| .png / .jpg | 스크린샷, 와이어프레임, 목업 |
| .xlsx | 스프레드시트, 예산, 로드맵 |
| .txt | 원본 텍스트, 로그 |

**팁:** PDF나 이미지의 경우 텍스트 인식이 주로 이루어지므로, 고품질 이미지 사용을 권장합니다.

## 4. 멀티 채널 출력 패턴: 단일 컨텍스트 → 다중 형식 변환

### 사용 사례: "주간 업데이트를 3가지 채널에 맞게 변환"

**입력:**
```
> @weekly-update.md 이 내용을 다음 3가지 형태로 변환해줘:
  1. Slack 메시지 (6줄 이내, 진행/리스크/요청)
  2. Email (제목 + 본문 7줄 + 요청사항 2개)
  3. Notion 페이지 (Summary/Metrics/Risks/Next Action)
```

**claude Code가 반환:**

```markdown
## 1️⃣ Slack 메시지

✅ 진행 중: 모바일 앱 베타 테스트 50명 모집 완료 (회원가입 폭주, 일부 서버 지연 발생)
⚠️ 리스크: 결제 게이트웨이 성능 이슈로 3명 가입 포기 (Stripe 지원 대기 중)
❓ 요청: 디자인팀 → 온보딩 UI 피드백 (내일 오후 3시까지)

---

## 2️⃣ Email

**제목:** [주간 보고] 모바일 앱 베타 - 50명 모집 완료, 결제 게이트웨이 최적화 필요

**본문:**

안녕하세요,

이번주 모바일 앱 베타 테스트를 위한 초기 50명 모집을 완료했습니다.
회원가입 폭주로 기대 이상의 관심을 확인했으나, Stripe 결제 게이트웨이 성능 저하로
3명의 사용자가 가입 중단했습니다. 스트라이프 지원팀과 협력하여 개선 중입니다.

현재까지 사용자 만족도 NPS는 72점(호평)이며, 온보딩 UI 개선에 대한 피드백을
디자인팀으로부터 수집 중입니다.

계획대로 진행 중이며, 내주 일요일까지 주요 성능 이슈 해결 예정입니다.

감사합니다.

**요청사항:**
- [긴급] 결제 게이트웨이: Stripe 기술팀과 협력
- [일반] 온보딩 UI 피드백: 내일 오후 3시까지 검토

---

## 3️⃣ Notion 페이지

### 📋 Summary
모바일 앱 베타 테스트 1주차 완료. 사용자 반응 긍정적(NPS 72)이나,
결제 게이트웨이 성능 이슈로 3명 이탈. 이번주 해결 목표.

### 📊 Metrics
| 항목 | 수치 | 목표 |
|------|------|------|
| 베타 모집 | 50명 | 50명 ✅ |
| NPS | 72 | 70+ ✅ |
| 완료 가입률 | 94% | 95% ⚠️ |

### 🚨 Risks
1. [높음] Stripe 게이트웨이 응답시간 8초 → 권장 < 2초
   - 영향: 결제 포기율 6%
   - 완화: Stripe 기술팀 에스컬레이션, 로컬 캐시 최적화

### 🎯 Next Action
- [ ] Stripe 성능 최적화 (마감: 내주 금요일)
- [ ] 온보딩 UI 피드백 통합 (마감: 내주 월요일)
- [ ] 베타 모집 2차 (50명 추가) 시작
```

**핵심 이점:**
- 소스는 1번만 입력 → 자동 포맷 변환
- 채널별 톤/스타일 자동 조정 (Slack: 캐주얼, Email: 정중, Notion: 구조화)
- 불일치 가능성 제거 (버전 통제 용이)

## 5. PM 팁: 입력 품질 = 출력 품질

### 입력 체크리스트

✅ **좋은 입력:**
- 명확한 목표: "이탈 원인을 찾아" vs "데이터를 봐"
- 필요한 맥락: 타겟층, 기간, 성공 기준을 포함
- 구조화된 데이터: 엑셀, CSV보다는 깔끔한 형식 선호

❌ **나쁜 입력:**
- 모호한 질문: "어떻게 생각해?"
- 누락된 배경: "지표가 떨어졌어" (언제? 얼마나?)
- 비정렬 파일: 한글/영문 섞임, 형식 일관성 없음

### 실전 팁

1. **파일 이름을 명시적으로:** `report.csv` → `2026q1-sales-by-region.csv`
2. **필요한 컨텍스트를 주석으로:** CSV 헤더 설명, 단위 명시
3. **예상 산출물을 명확히:** "차트 3개" 또는 "액션 플랜"
4. **마지막에 검증 요청:** "이게 맞는지 원본 파일로 다시 한 번 점검해줄래?"

## 6. PM 역할 확장 포인트

### Before: 기존 방식
| 작업 | 소요 시간 | 방법 |
| --- | --- | --- |
| 데이터 분석 요청 | 2~3일 | 데이터팀에 Jira 티켓 생성, 대기 |
| 경쟁사 앱 스크린샷 분석 | 2시간 | 수동으로 캡처, PPT에 정리 |
| 주간 업데이트 3채널 전달 | 1시간 | Slack/Email/Notion에 각각 작성 |

### After: Claude Code 활용
| 작업 | 소요 시간 | 방법 |
| --- | --- | --- |
| 데이터 분석 | 10분 | `@data.csv 이탈 원인 분석해줘` 직접 실행 |
| 경쟁사 앱 스크린샷 분석 | 5분 | Ctrl+V → "UX 문제점 3개 찾아줘" |
| 주간 업데이트 3채널 전달 | 3분 | 1회 입력 → 자동 포맷 변환 |

### 역할 변화의 본질
- **자동화**: 포맷 변환, 반복 작업 → Claude가 처리
- **증강**: 데이터 분석, 패턴 인식 → Claude가 초안, PM이 판단
- **직접 판단**: "이 인사이트가 우리 로드맵에 어떤 의미인가?" → PM만 가능

## 7. 실습 과제

1. 자신의 프로젝트에서 CSV 파일 1개 선택 → Claude Code에 `@파일명 이 데이터의 핵심 3가지 인사이트를 찾아줘` 실행
2. 스크린샷 1개 캡처 → Ctrl+V로 붙여넣기 → "이상 징후가 있는지 봐줘" 요청
3. 마크다운 문서 1개 → 이를 Slack, Email, Notion 3가지로 변환 요청
