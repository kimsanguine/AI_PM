# 8.2 AI 네이티브 PM으로의 성장 경로

## 역량 레벨별 학습 로드맵

모든 PM이 한 번에 모든 것을 배울 필요는 없습니다. 자신의 레벨과 현재 역할에 맞는 경로를 선택하세요.

---

## Junior PM: Part 0~1 (입문 단계)

**목표**: Claude Code의 기본 조작 숙련 및 반복 업무 자동화

### 학습 내용
- Claude Code 기본 조작 (파일 참조, 모드 전환)
- 간단한 분석 (CSV 읽기, 텍스트 정리)
- 문서 작성 자동화 (회의록, 이메일 작성)
- 데이터 요약 (대량 텍스트 → 핵심 정리)

### 실무 적용 예시

#### 회의록 정리 (30분 → 5분)

```
Before:
1. 회의 녹음 듣기 (30분)
2. 수동으로 회의록 작성 (20분)
3. 액션 아이템 정리 (10분)
총 60분

After:
1. 회의 음성 텍스트 변환 (자동)
2. 텍스트를 Claude에 업로드
   > 회의 내용을 정리해줘.
     - 주요 결정사항
     - 액션 아이템 (담당자, 기한)
     - 다음 미팅 날짜
3. 결과물 편집 (5분)
총 10분
```

#### 주간 업데이트 작성 (2시간 → 30분)

```
Before:
- 각자가 개별적으로 진행 상황 정리
- PM이 수동으로 통합 및 포맷팅
- 여러 번 왕복 수정

After:
> 다음 정보를 주간 업데이트 형식으로 정리해줘:
  - WAT: 2,100팀 (+5%)
  - Activation: 54% (+2pp)
  - 주요 이슈: [리스트]
  - 다음주 계획: [리스트]

Claude가 즉시 전문적 포맷의 문서 생성
```

### 예상 학습 시간
- 기본 개념: 2시간
- 실습 및 적용: 2~4시간
- **총 4\~6시간**

### 성과 지표
- 주간 회의록 작성 시간 80% 단축
- 데이터 정리 업무 50% 자동화
- 보고서 작성 시간 60% 단축

## Practitioner PM: Part 2~3 (실무 단계)

**목표**: Discovery & Definition 단계에 Claude Code 적극 활용, 팀 협업 확장

### 학습 내용
- PRD 작성 자동화 (구조화된 문서 생성)
- 경쟁사 분석 (업체 정보 → 대비표 자동 생성)
- 사용자 리서치 정리 (인터뷰 노트 → 요약 및 테마 추출)
- 커스텀 서브에이전트 설계 (반복되는 분석 패턴 자동화)

### 실무 적용 예시

#### PRD 작성 (1주 → 2일)

```
Before:
- 요구사항 수집 및 정리 (2~3일)
- PRD 문서 작성 (1~2일)
- 이해관계자 피드백 및 수정 (1~2일)
총 5~7일

After:
> 다음 기능의 PRD를 작성해줘:
  - 기능명: 팀 멤버 초대
  - 배경: [상황 설명]
  - 목표: [구체적 지표]
  - 요구사항:
    * 이메일/링크로 초대
    * 권한 설정 (Admin/Editor/Viewer)
    * 초대 상태 추적
  - 세부 사항: [흐름도]

Claude가 구조화된 PRD 초안 생성 (30분)
  → 이해관계자 피드백 (1일)
  → PM이 수정 반영 (1일)
총 2~3일
```

#### 경쟁사 분석 (3일 → 4시간)

```
Before:
- 경쟁사 10개 웹사이트 방문
- 가격, 기능, 타겟 고객 수동 정리 (2~3시간)
- 스프레드시트 작성 (1시간)
- 분석 및 인사이트 도출 (1시간)
총 4~5시간

After:
> 경쟁사 분석 리포트를 만들어줘:
  대상: Slack, Microsoft Teams, Asana
  항목: 가격 모델, 핵심 기능, 타겟, 강점/약점

Claude Code:
1. 각 웹사이트 정보 수집
2. 비교 테이블 자동 생성
3. 우리의 차별화 전략 제안

결과물 (4시간)
- 상세 비교표
- 강점/약점 분석
- 포지셔닝 제안
```

#### 사용자 리서치 테마 추출 (8시간 → 1시간)

```
Before:
- 10개 인터뷰 노트 수동 읽기
- 공통 주제/패턴 식별 (6~7시간)
- 개선 제안 정리 (1~2시간)
총 8~9시간

After:
> 사용자 인터뷰 노트 10개를 분석해줘:
  @interview_1.txt @interview_2.txt ...

  분석 항목:
  - 주요 문제점 (빈도 순)
  - 사용자 유형별 이슈 차이
  - 개선 요청 우선순위
  - 기회 영역 식별

Claude가 자동 분석 및 테마화 (30분)
  → PM 최종 검토 (30분)
총 1시간
```

### 커스텀 서브에이전트 설계

반복되는 분석 작업을 자동화:

```
예: "경쟁사 분석 에이전트"

매주 설정된 경쟁사 목록을 자동 모니터링
→ 가격 변경, 새 기능 추가 감지
→ 변경사항 정리하여 Slack 알림
→ 월간 리포트 자동 생성

이렇게 설정하면, PM은 매주 자동화된 리포트만 검토
```

### 예상 학습 시간
- Part 0~1 기초: 4~6시간
- Part 2~3 심화: 2~3시간
- **총 6\~8시간**

### 성과 지표
- PRD 작성 시간 70% 단축
- 경쟁사 분석 주기 단축 (1개월 → 1주)
- 사용자 리서치 인사이트 도출 시간 90% 단축

## Lead PM / Product Lead: Part 4~5 (리더십 단계)

**목표**: 제품 전략 수립 및 팀 역량 확대, AI 기반 의사결정 주도

### 학습 내용
- 바이브 코딩으로 프로토타입 직접 제작 (4.1절)
- 시각적 에셋 생성 (4.2절)
- 배포 및 라이브 모니터링 (4.3절)
- A/B 테스트 분석 및 의사결정 (5.1절)
- KPI 체계 설계 및 자동화 (5.2절)
- 팀에 Claude Code 도입 리딩

### 실무 적용 예시

#### 프로토타입 검증 루프 (1주 → 1일)

```
Before:
- 아이디어 정의 (1일)
- 설계 문서 작성 (1일)
- 개발팀 브리핑 (0.5일)
- 개발 (3~5일)
- 첫 검증 미팅 (1일)
총 7~10일

After:
- PM이 의도 설명 (30분)
- Claude Code로 프로토타입 생성 (3~4시간)
- 스크린샷 기반 피드백 반복 (3~4시간)
- 당일 또는 다음날 검증 완료
총 1~2일
```

#### 데이터 기반 의사결정 (1주 → 1일)

```
Before:
- A/B 테스트 실행 (2주)
- 데이터팀에 분석 요청 (1일)
- 분석 완료 대기 (3~5일)
- 결과 리포트 (1일)
- 의사결정 회의 (1일)
총 3~4주

After:
- A/B 테스트 실행 (2주)
- PM이 Claude로 1차 분석 (1시간)
- 데이터팀과 심화 논의 (2시간)
- 의사결정 (즉시)
총 2주 + 1일
```

### 팀 도입 가이드 (Lead 역할)

```
Phase 1: 준비 (1주)
- 팀 전체에 Claude Code 소개 (30분 워크샵)
- 각 팀원별 사용 시나리오 공유
- 보안, 윤리 가이드라인 수립

Phase 2: 파일럿 (2주)
- 2~3명 선발팀이 먼저 사용해보기
- 일일 동기화 (15분): 배운 점, 어려움 공유
- 팁 및 모범 사례 정리

Phase 3: 전사 확대 (2주)
- 전체 팀 온보딩 및 교육
- 일주일 집중 사용 (회의록, 데이터 정리 등)
- 피드백 수집 및 개선

Phase 4: 고도화 (지속)
- 팀별 커스텀 워크플로 개발
- 주간 "Claude Code 팁" 공유
- 분기별 선진 사례 공유 (MCP, n8n 연동 등)
```

### 예상 학습 시간
- Part 0~1 기초: 4~6시간
- Part 2~3 심화: 2~3시간
- Part 4~5 리더십: 4~5시간
- **총 10\~12시간**

### 성과 지표
- 프로토타입 검증 시간 80% 단축
- 의사결정 시간 70% 단축
- 팀 생산성 30% 향상 (6개월 기준)

## 성장 경로 선택 가이드

### 당신의 상황은?

#### "나는 막 PM이 되었거나, 운영 업무가 많아요"
→ **Junior 경로 추천**
- 회의록, 보고서 작성 자동화부터 시작
- 4~6시간 투자로 즉시 생산성 향상
- 3개월 후: 주 5시간 절감 가능

---

#### "나는 경력 PM이고, 더 전략적으로 일하고 싶어요"
→ **Practitioner 경로 추천**
- 분석, 리서치, 문서화를 AI와 협업
- 6~8시간 투자로 전략적 사고에 더 많은 시간 확보
- 3개월 후: 주 12시간 절감 가능

---

#### "나는 Product Lead / 디렉터이고, 팀을 리딩하고 있어요"
→ **Lead 경로 추천**
- 직접 프로토타입 제작 및 배포
- 데이터 기반 의사결정 주도
- 팀 역량 강화
- 6개월 후: 제품 주기 30% 단축, 팀 생산성 30% 향상

## 다음 단계: 고급 자동화 확장

모든 레벨을 마친 후, 추가로 학습할 수 있는 고급 주제들:

### MCP (Model Context Protocol) 연동
- 실시간 데이터베이스 쿼리
- Slack, Google Sheets 등 도구와 자동 연동
- 예: 매일 자동으로 KPI 대시보드 업데이트

### n8n 워크플로 자동화
- 복잡한 다단계 작업 자동화
- 예: 주간 KPI 리포트 자동 생성 및 발송
- 예: A/B 테스트 결과 자동 분석 및 경보

### Agentic AI 설계
- Claude가 스스로 판단하여 행동하는 에이전트
- 예: "매주 이 분석을 수행하고 결과를 Slack으로 보내"
- 예: "경쟁사 가격 변동을 감시하고, 20% 이상 변경 시 알려줘"

## PM으로서의 마인드셋 변화

### FROM "코딩은 개발자가 하는 것"
### TO "나도 빠르게 구현할 수 있다"

```
이전: 아이디어 → 설계 → 요청 → 대기 (1~2주)
현재: 아이디어 → 검증 (당일)

결과:
- 실패 비용 감소 (검증 후 개발)
- 반복 속도 증가 (피드백 루프 1주 → 1일)
- 제품 오너십 강화 (직접 만들고 검증)
```

### FROM "데이터 분석은 분석가가 하는 것"
### TO "나도 1차 분석을 할 수 있다"

```
이전: 데이터 요청 → 대기 (3~5일) → 분석 결과 받음 → 의사결정
현재: 데이터 → 즉시 분석 (1시간) → 의사결정

결과:
- 의사결정 속도 급증 (주 단위 → 일 단위)
- 분석가는 더 깊은 분석에 집중
- 데이터 리터러시 향상
```

### FROM "문서는 타이핑으로"
### TO "구조화된 사고를 AI가 문서로"

```
이전: 직접 타이핑 (2~3시간) → 여러 번 수정
현재: 아이디어만 설명 (30분) → AI가 초안 작성 (30분) → 검토

결과:
- 창의력에 시간 사용 (쓰기 ≠ 생각)
- 문서 품질 향상 (구조화)
- 커뮤니케이션 명확성 증가
```
