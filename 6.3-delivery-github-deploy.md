# 4.3 GitHub & 배포: 아이디어를 세상에 공개하기

## Git 기본: PM이 알아야 할 최소한

PM이 코드를 이해할 필요는 없지만, **버전 관리의 개념**과 **배포 워크플로우**는 이해해야 합니다.

### Git이란?
- 코드 변화 이력 기록 시스템
- "저장(Commit)", "공개(Push)", "협업(Merge)" 의 개념 이해만으로 충분

### GitHub란?
- Git 저장소를 온라인에서 공유하는 플랫폼
- 협업, 코드 리뷰, 배포의 중심

---

## 실전: 성격 유형 테스트를 GitHub에 올리기

### 1단계: 저장소 생성 및 초기 커밋

```bash
$ cd ~/my-prototype/personality-quiz
$ claude
```

Claude와의 대화:

```
> 지금까지의 변경사항을 GitHub에 올려줘.
  저장소 이름은 "personality-quiz"로.
  커밋 메시지는 "feat: 성격 유형 테스트 MVP 완성"으로.

Claude: GitHub 저장소를 생성하고 코드를 푸시합니다.

✓ Git 저장소 초기화 (git init)
✓ .gitignore 파일 생성 (node_modules, .next, .env.local 제외)
✓ 모든 파일 스테이징 (git add .)
✓ 초기 커밋 생성
  Commit: feat: 성격 유형 테스트 MVP 완성
  Files changed: 23 files
  Insertions: 1,247 lines

✓ GitHub 저장소 생성 중...
✓ 원격 저장소 설정 (git remote add origin)
✓ main 브랜치로 푸시 (git push -u origin main)

Repository created: https://github.com/username/personality-quiz
```

### GitHub 저장소 구조

```
personality-quiz/
├── README.md              # 프로젝트 설명
├── package.json          # 패키지 의존성
├── tsconfig.json         # TypeScript 설정
├── pages/
│   ├── index.tsx         # 시작 페이지
│   ├── quiz.tsx          # 퀴즈 페이지
│   └── result.tsx        # 결과 페이지
├── components/
│   └── Question.tsx
├── lib/
│   ├── personalities.ts  # 성격 유형 데이터
│   └── scoring.ts        # 점수 계산
├── public/
│   └── [이미지 자산들]
└── .gitignore
```

### 2단계: 변경사항 커밋 (반복)

개발 과정에서 주요 기능 완성마다 커밋:

```
> 결과 공유 기능을 완성했어. 커밋해줘.
  메시지: "feat: 결과 공유 기능 추가"

Claude: 변경사항을 커밋합니다.

✓ pages/result.tsx 수정
✓ lib/sharing.ts 생성
✓ Commit: feat: 결과 공유 기능 추가
✓ Git push 완료

GitHub 커밋 링크:
https://github.com/username/personality-quiz/commit/a1b2c3d
```

### 3단계: 협업 (브랜치 사용)

대규모 수정이나 실험 기능:

```
> 데이터 분석 기능을 시험해보고 싶어.
  별도 브랜치에서 작업해줄 수 있어?

Claude: 새로운 브랜치를 생성합니다.

✓ 브랜치 생성: feature/analytics
✓ 브랜치 전환 (git checkout feature/analytics)
✓ 코드 작성 및 커밋

나중에 확인 후:
> 결과가 좋으니 main 브랜치에 병합해줘.

Claude: 브랜치를 병합합니다.

✓ main 브랜치로 전환
✓ feature/analytics 병합
✓ 충돌 자동 해결
✓ Git push 완료
```

## Vercel 배포: 누구나 접근 가능한 라이브 서비스

**Vercel**은 Next.js 프레임워크 제작사의 호스팅 플랫폼입니다. 클릭 몇 번으로 배포 완료.

### 배포 프로세스

```
> 이 프로젝트를 Vercel에 배포해줘.
  URL은 personality-quiz.vercel.app으로.

Claude: Vercel 배포를 설정합니다.

✓ Vercel 계정 설정 확인
✓ GitHub 연결 (OAuth)
✓ 프로젝트 임포트
✓ 배포 설정:
  - Build Command: npm run build
  - Output Directory: .next
  - Environment Variables: 없음

✓ 초기 배포 시작 중...
✓ 빌드 완료 (2분 17초)
✓ 배포 완료

Live URL: https://personality-quiz.vercel.app
```

### 배포 후 실시간 모니터링

```
> 배포된 사이트에 접속해서 정상 작동하는지 확인해줘.

Claude: 라이브 테스트를 수행합니다.

✓ 랜딩 페이지 로드: 성공
✓ 퀴즈 시작: 성공
✓ 질문 5개 완료: 성공
✓ 결과 페이지 표시: 성공
✓ 결과 공유 URL 생성: 성공
✓ 공유 링크 접속: 성공

모든 주요 기능이 정상 작동합니다.

배포 대시보드: https://vercel.com/dashboard
```

### 자동 배포 (CI/CD)

GitHub에 코드를 푸시하면 자동으로 배포:

```
main 브랜치에 푸시
  ↓
Vercel에서 자동 감지
  ↓
빌드 프로세스 실행 (2~3분)
  ↓
테스트 수행
  ↓
라이브 배포 완료
  ↓
Slack 알림 (자동 설정 시)
```

## PM의 역할: 배포 후 48시간 모니터링

배포 완료 후 PM이 집중해야 할 업무:

### 1. 사용자 피드백 수집 (24시간)

```
배포 후 체크리스트:
□ 내부 팀 QA (30분)
  - 주요 기능 재확인
  - 엣지 케이스 테스트 (빈 입력, 부정확한 데이터)

□ 초기 사용자 테스트 (스팀 멤버, 베타 유저 대상)
  - 사용성 피드백
  - 에러 리포트

□ 분석 데이터 수집 (24시간 경과)
  - 접속 수, 이탈률
  - 주요 사용 경로
  - 오류 발생 패턴
```

### 2. 핫픽스 준비

만약 심각한 버그 발견:

```
> 결과 페이지에서 공유 버튼이 작동하지 않아.
  급히 수정해서 배포해야 해.

Claude: 핫픽스를 진행합니다.

✓ 버그 원인 파악: pages/result.tsx의 onClick 핸들러
✓ 코드 수정
✓ 로컬 테스트
✓ 커밋: hotfix: 공유 버튼 버그 수정
✓ GitHub 푸시
✓ Vercel 자동 배포 시작

✓ 배포 완료 (1분 42초)
New URL: https://personality-quiz.vercel.app (ver 1.1)
```

### 3. 피드백 기반 이터레이션

48시간 모니터링 후:

```
수집된 피드백:
- 결과 공유율: 34% (목표 40%)
- 질문 3에서 이탈률 높음 (22% → 28%)
- 모바일 UX 불편 (폰트 작음)
- 사용자 요청: 결과 재시도 기능

다음 이터레이션 계획:
□ 질문 3 텍스트 간소화
□ 모바일 폰트 크기 20% 증가
□ "다시 시도" CTA 개선
```

## PM 역할 확장 포인트

### BEFORE: 배포는 엔지니어 전담
```
아이디어/프로토타입 완성
  ↓
개발 팀에 배포 요청
  ↓
배포 설정 (1~2일)
  ↓
배포 실행 및 모니터링
  ↓
문제 발생 시 개발 팀 호출

PM의 역할: 수동 대기 → 배포 후 분석만 담당
```

**문제점**: 프로토타입 검증 지연, 피드백 루프 느림

---

### AFTER: PM이 직접 배포
```
아이디어/프로토타입 완성
  ↓
PM이 GitHub에 푸시 & Vercel 배포 (30분)
  ↓
배포 후 실시간 모니터링 (48시간)
  ↓
데이터 수집 및 피드백 분석
  ↓
즉시 다음 이터레이션 계획

PM의 역할: 능동적 배포 관리 → 빠른 검증 주기
```

**이점**:
- **배포 시간**: 1~2일 → 30분 단축
- **피드백 루프**: 1주 → 24시간 단축
- **의사결정 속도**: 데이터 기반의 빠른 선택
- **제품 오너십**: 더 강한 주인의식 강화

**배포 권한 자동화**: GitHub Push → Vercel 배포 (수동 개입 불필요)

## 배포 후 일반적인 질문

### Q1: 프로덕션에서 오류 발생 시?
```
> 라이브 사이트에서 JavaScript 에러가 발생했어.

Claude: 로그를 확인하고 핫픽스를 진행합니다.
- Vercel 대시보드에서 에러 로그 검토
- 원인 파악 및 수정
- 즉시 배포
- 유저 영향도 분석
```

### Q2: 예전 버전으로 롤백하려면?
```
> 어제 버전으로 되돌려줄 수 있어?

Claude: Vercel 대시보드에서 이전 배포로 되돌립니다.
✓ 배포 이력 확인
✓ 원하는 버전 선택
✓ 즉시 롤백 (1분 이내)
```

### Q3: 프라이빗 데이터 관리는?
```
데이터베이스 연결, 환경 변수는
Vercel의 암호화된 설정(Environment Variables)에서 관리.
민감한 정보는 GitHub에 저장되지 않습니다.
```
