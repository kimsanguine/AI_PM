# 유즈케이스: 비즈니스 케이스 / 투자 메모 작성

> **PM 역량**: 전략적 사고 / 재무 모델링 / 이해관계자 설득
> **난이도**: P → L
> **관련 문서**: [5.2-definition-product-strategy.md](./5.2-definition-product-strategy.md), [A.3-usecase-scenarios.md](./A.3-usecase-scenarios.md)

---

## 왜 PM에게 필요한가?

PM이 새 기능이나 제품 라인을 제안할 때, 가장 큰 허들은 "왜 이것에 투자해야 하는가?"에 대한 설득력 있는 답변입니다. 좋은 PRD가 있어도, 경영진이 궁금한 것은 "시장 기회는 얼마인가?", "ROI는?", "리스크는?" 같은 투자 관점의 질문입니다.

투자 메모(Investment Memo) 또는 비즈니스 케이스는 이런 질문에 체계적으로 답하는 문서입니다. Claude Code는 **시장 데이터 수집 → 재무 모델링 → 리스크 분석 → 문서 작성**을 지원합니다.

---

## 상황

당신은 AI Avatar 서비스의 PM입니다. "AI 아바타 기반 실시간 고객 응대 서비스"라는 새 제품 라인을 제안하려고 합니다. 다음 이사회에서 $2M 투자 승인을 받아야 합니다.

---

## 기존 방식 vs Claude 방식

```
기존 (2~3주):
1. 시장 리서치 (3~5일)
2. 경쟁 분석 (2일)
3. 재무 모델 스프레드시트 작성 (3일)
4. 시나리오 분석 (2일)
5. 투자 메모 문서 작성 (3일)
6. 리뷰 및 수정 (2~3일)

Claude (3~4일):
1. 데이터 수집 + 시장 분석 초안 (반일)
2. 재무 모델 초안 + 시나리오 (반일)
3. PM 검증 + 내부 데이터 보강 (1일)
4. 투자 메모 초안 작성 (2시간)
5. 리뷰 + 질의응답 시뮬레이션 (1일)
```

## 워크스루

### Step 1: 투자 기회 프레이밍

```
Terminal 입력:
> "AI 아바타 기반 실시간 고객 응대 서비스" 투자 메모를 작성하려고 해.

  ## 투자 메모 구조
  1. Executive Summary (1페이지)
  2. Market Opportunity (시장 규모, 트렌드, 타이밍)
  3. Product Vision (무엇을, 왜, 어떻게)
  4. Competitive Landscape (진입 장벽, 차별화)
  5. Financial Model (투자 규모, ROI, 회수 기간)
  6. Risk Analysis (리스크 매트릭스, 완화 전략)
  7. Ask (구체적 요청사항)

  ## 맥락
  - 우리는 이미 비동기 아바타 영상 서비스를 운영 중 (DAU 12K)
  - 실시간 대화형 아바타는 새로운 기술 스택 필요
  - 투자 요청액: $2M (12개월)
  - 목표: MVP 6개월, 유료 전환 12개월

  우선 Market Opportunity 섹션부터 작성해줘.
  각 수치에 [소스]와 [가정]을 명확히 구분해서.

Claude Code 응답:

# 2. Market Opportunity

## 2.1 시장 규모

| 구분 | 규모 | 소스 |
|------|------|------|
| 글로벌 대화형 AI 시장 | $15.7B (2025) | [Grand View Research, 2024] |
| 연간 성장률 (CAGR) | 23.6% | [Markets & Markets, 2024] |
| AI 아바타 세그먼트 | $2.1B (2025) | [가정: 대화형 AI의 13.4%] |
| 실시간 아바타 응대 | $420M (2025) | [가정: AI 아바타의 20%] |

⚠️ [가정 투명성]
- "AI 아바타 세그먼트 13.4%" 근거:
  Gartner의 카테고리 분류에서 Virtual Agent 비중
- "실시간 20%" 근거:
  현재 실시간 비율이 낮지만 성장률이 비동기 대비 2x
  → 2027년에는 35%로 확대 예상

## 2.2 타이밍이 맞는 이유

1. **기술 성숙도**: 실시간 아바타 렌더링이
   상용 수준에 도달 (NVIDIA ACE, 자체 모델)
   [근거: NVIDIA GTC 2024 벤치마크]

2. **고객 수요 신호**: 기존 고객 Enterprise 문의 중
   "실시간 가능한가?" 질문 비율
   - 2024 Q2: 8%
   - 2024 Q4: 23% (↑ 188%)
   [근거: Hubspot 딜 노트 분석]

3. **경쟁 공백**: HeyGen Interactive Avatar 출시했으나
   Enterprise 보안 요구사항 미충족
   [근거: G2 리뷰 + Enterprise RFP 분석]
```

### Step 2: 재무 모델

```
Terminal 입력:
> Financial Model 섹션을 작성해줘.
  3가지 시나리오를 만들어야 해:

  1. Base Case: 보수적 가정
  2. Upside Case: 낙관적 가정 (기존 고객 전환 + 신규 수요)
  3. Downside Case: 비관적 가정 (기술 지연, 시장 느림)

  각 시나리오별로:
  - 12개월 비용 구조 (인력, 인프라, 마케팅)
  - 매출 예측 (고객 수 × ARPU)
  - 회수 기간 (Payback Period)
  - 3년 NPV

  중요: 모든 가정에 [근거] 또는 [PM 판단]을 표시해.
  재무팀이 볼 때 "이 숫자는 어디서 왔는가?"에 답할 수 있어야 해.

Claude Code 응답:

# 5. Financial Model

## 5.1 투자 비용 구조 (12개월)

| 항목 | 금액 | 비율 | 근거 |
|------|------|------|------|
| 엔지니어링 인력 (6명) | $1,080K | 54% | [시장 평균 연봉 기반] |
| GPU 인프라 | $480K | 24% | [AWS g5 인스턴스 × 예상 트래픽] |
| 제품/디자인 (2명) | $240K | 12% | [현 팀 리소스 재배치] |
| 마케팅/GTM | $120K | 6% | [기존 채널 활용, 최소 투자] |
| 버퍼 | $80K | 4% | [표준 10% → 4%로 축소, 근거 아래] |
| **합계** | **$2,000K** | **100%** | |

⚠️ [PM 판단 필요]
- 버퍼 4%는 공격적입니다. 통상 10~15% 권장
- 다만 기존 인프라 활용으로 리스크가 낮은 항목이 많아
  4%로 설정. 이사회에서 질문받을 수 있으니 10% 버전도 준비

## 5.2 매출 예측

### Base Case
| 지표 | Month 6 (MVP) | Month 12 | 근거 |
|------|---------------|----------|------|
| 파일럿 고객 | 5 | 15 | [기존 Enterprise 중 관심 표명 23%] |
| ARPU | $2,000 | $3,500 | [비동기 Enterprise ARPU의 1.5~2x] |
| MRR | $10K | $52.5K | 고객 × ARPU |
| ARR (Run-rate) | $120K | $630K | MRR × 12 |
| 회수 기간 | — | 38개월 | 투자 $2M ÷ 연 $630K |

### Upside Case
| 지표 | Month 6 | Month 12 | 근거 |
|------|---------|----------|------|
| 파일럿 고객 | 8 | 30 | [기존 고객 전환 + 신규 인바운드] |
| ARPU | $2,500 | $4,500 | [프리미엄 포지셔닝 성공 시] |
| MRR | $20K | $135K | |
| ARR | $240K | $1,620K | |
| 회수 기간 | — | 15개월 | |

### Downside Case
| 지표 | Month 6 | Month 12 | 근거 |
|------|---------|----------|------|
| 파일럿 고객 | 2 | 8 | [기술 지연 3개월, MVP 9개월] |
| ARPU | $1,500 | $2,000 | [가격 경쟁 압박] |
| MRR | $3K | $16K | |
| ARR | $36K | $192K | |
| 회수 기간 | — | 125개월 ❌ | → Pivot 필요 신호 |

## 5.3 의사결정 프레임워크

Base Case 기준:
- 38개월 회수는 B2B SaaS 표준 (24~36개월) 대비 보수적
- 하지만 기존 고객 기반 활용 + 전략적 가치 고려 시 합리적

⚠️ [Kill Criteria 제안]
Month 6 시점 파일럿 고객 < 3이면 Pivot 또는 중단 검토
→ 이사회에 "명확한 종료 기준을 가지고 투자한다"는 메시지
```

### Step 3: 이사회 Q&A 시뮬레이션

```
Terminal 입력:
> 이사회에서 받을 가능성이 높은 질문 Top 10을 예상하고,
  각 질문에 대한 답변 초안을 만들어줘.

  특히 다음 유형의 질문에 대비해:
  1. 재무적 질문 (ROI, 회수 기간, 기회비용)
  2. 전략적 질문 (왜 지금? 왜 우리?)
  3. 리스크 질문 (가장 큰 리스크는? 실패하면?)
  4. 실행 질문 (팀은? 일정은? 마일스톤은?)

Claude Code 응답:

## 예상 질문 & 답변 시뮬레이션

### Q1: "38개월 회수면 너무 긴 것 아닌가?"
**답변**: "Base Case 38개월은 보수적 가정입니다. 현실적으로는
기존 고객 전환을 고려하면 18~24개월이 더 가능성이 높습니다.
또한 이 투자는 단순 ROI 외에 전략적 가치가 있습니다:
(1) 경쟁사가 이 시장에 진입하기 전 선점 효과
(2) 기존 비동기 고객의 확장 매출 (Expansion Revenue)
(3) Enterprise 솔루션 포지셔닝 강화"

### Q2: "HeyGen이 이미 실시간 아바타를 출시했는데?"
**답변**: "HeyGen Interactive Avatar는 SaaS 전용이며
Enterprise 보안 요구사항을 충족하지 못합니다. 우리의 On-premise
배포 역량은 금융/의료 분야에서 결정적 차별화입니다.
G2 리뷰에서도 HeyGen의 Enterprise 지원에 대한 불만이
꾸준히 올라오고 있습니다."

### Q3: "실패하면 $2M을 날리는 건가?"
**답변**: "Kill Criteria를 설정합니다. Month 6에 파일럿 고객 3곳
미만이면 즉시 Pivot/Stop 검토합니다. 이 경우 소진 비용은
약 $800K이며, 개발된 실시간 렌더링 기술은 기존 비동기 제품의
품질 향상에 재활용 가능합니다."
...

⚠️ [PM 확인 필요]
Q&A 시뮬레이션은 일반적 이사회 패턴 기반입니다.
귀사 이사회의 특성 (기술 지향 vs 재무 지향, 주요 보드 멤버의
관심사)에 맞게 PM이 우선순위를 조정하세요.
```

## PM 판단 포인트

### ❓ "Claude가 만든 재무 모델을 믿어도 되나?"

```
Claude가 잘하는 것:
- 시장 데이터 수집 및 구조화
- 시나리오별 모델 틀 잡기
- 가정과 근거의 체계적 분리
- Q&A 시뮬레이션 (일반적 패턴)

PM이 반드시 직접 해야 하는 것:
1. [내부 데이터] 실제 ARPU, CAC, Churn은 내부 데이터에서
2. [가정 검증] "기존 고객 23% 전환"이 현실적인지
   → 실제 세일즈팀에 확인 필요
3. [정치적 프레이밍] 이사회 멤버 성향에 맞는 톤 조절
4. [Kill Criteria] "실패 기준"을 누가, 어떻게 판단할지
5. [기회비용] "$2M을 다른 곳에 쓰면?"에 대한 비교 분석
```

### ❓ "가정(Assumption)과 사실(Fact)을 어떻게 구분하나?"

```
투자 메모에서 가장 중요한 것은 투명성입니다.

[Fact]: 공개 데이터로 검증 가능한 수치
→ "글로벌 대화형 AI 시장 $15.7B" — 리서치 리포트 인용

[Estimate]: 합리적 추정이 가능한 수치
→ "AI 아바타 세그먼트 비중 13.4%" — 카테고리 추론

[Assumption]: PM의 판단이 포함된 수치
→ "파일럿 고객 15곳" — 현재 파이프라인 + 전환율 추정

이사회는 [Assumption]에서 집중 질문합니다.
가정의 근거가 탄탄할수록 신뢰도가 올라갑니다.
```

## 실습 과제

### Level 1 (J): 미니 비즈니스 케이스
기존 제품의 기능 하나(개발 1개월 규모)에 대해 비즈니스 케이스를 작성하세요. 시장 기회, 비용, 예상 효과를 포함하세요.

### Level 2 (P): 투자 메모 + Q&A 대비
$500K~$2M 규모의 새 이니셔티브에 대한 투자 메모를 작성하세요. 3가지 시나리오 모델 + 예상 질문 10개 답변을 준비하세요.

### Level 3 (L): 포트폴리오 투자 프레임워크
3개 이상의 투자 기회를 동시에 평가하는 프레임워크를 만드세요. 제한된 예산을 어떻게 배분할지를 데이터 기반으로 의사결정하세요.
