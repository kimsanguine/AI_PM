# 설치부터 첫 대화까지

## 사전 요구사항

Claude Code를 시작하기 전에 다음 세 가지가 필요하다:

### 1. Claude Pro 또는 Max 구독
Claude Code는 Claude API를 사용한다. 일반 무료 계정으로는 안 된다.
- **Claude Pro**: 기본 기능 (권장)
- **Claude Max**: 더 많은 컨텍스트와 빠른 처리 (선택사항)

[claude.ai](https://claude.ai)에서 구독을 시작할 수 있다.

### 2. Node.js 18 이상

Claude Code는 Node.js 위에서 동작한다. 설치 확인:
```
$ node --version
v18.17.0  # 이상이면 OK
```

아직 설치하지 않았다면:
- **macOS**: `brew install node`
- **Windows**: [nodejs.org](https://nodejs.org)에서 LTS 버전 다운로드
- **Linux**: `sudo apt install nodejs` (Ubuntu/Debian)

### 3. 터미널 접근
macOS나 Linux는 기본 터미널. Windows는 PowerShell 또는 Windows Terminal을 사용한다.

## 설치 명령어

설치는 플랫폼별로 다르다.

### macOS & Linux
터미널을 열고:
```
curl -fsSL https://claude.ai/install.sh | sh
```

이 명령어는:
1. Claude Code 바이너리를 다운로드
2. `/usr/local/bin` 또는 `~/.local/bin`에 설치
3. PATH에 등록

### Windows (PowerShell)
PowerShell을 관리자로 열고:
```
irm https://claude.ai/install.ps1 | iex
```

또는 WSL2 (Windows Subsystem for Linux)를 사용 중이면, macOS와 같은 방식:
```
curl -fsSL https://claude.ai/install.sh | sh
```

### 설치 완료 확인
어느 플랫폼이든, 설치 후:
```
$ claude --version
claude-code v1.2.4
```

버전 번호가 나오면 설치 완료다.

## 초기 설정

처음 Claude Code를 실행할 때, API 키를 요청받는다.

### 1단계: 첫 실행
아무 프로젝트 폴더에서나:
```
$ claude
```

### 2단계: 인증
그러면:
```
╭─────────────────────────────────────────────╮
│ Claude Code Setup                           │
│ API key가 필요합니다.                       │
│ claude.ai에서 발급받으세요.                 │
╰─────────────────────────────────────────────╯

Paste your API key:
```

https://platform.claude.com/ 에서 API 키를 생성하고, 터미널에 붙여넣는다.
위에서 Pro, Max 구독한 경우에는 구독을 선택하고 브라우저 화면의 단계를 거친다.

### 3단계: 상태 확인
API 키를 입력한 후:
```
$ claude /doctor

✓ Node.js 18+ detected (v18.17.0)
✓ Authentication valid
✓ Network connection OK
✓ Claude Code ready to use

API Key Status: Active
Model: claude-3-5-sonnet (default)
```

모든 항목이 ✓라면 설정 완료.

## 첫 대화 시작하기

### 프로젝트 폴더로 이동
Claude Code는 현재 폴더를 기준으로 파일을 읽고 쓴다. 따라서 프로젝트 폴더에서 시작하는 게 좋다:

```
$ cd ~/my-product  # 자신의 프로젝트 폴더
$ claude
```

### Claude와의 첫 대화
이제 프롬프트가 나타난다:
```
╭──────────────────────────────────────────────╮
│ ✻ Welcome to Claude Code!                   │
│   Type /help for available commands          │
│   /exit to quit                               │
╰──────────────────────────────────────────────╯

Project: my-product
Location: /Users/yourname/my-product

>
```

첫 질문을 해보자. 예를 들어, 프로젝트의 README가 있다면:

```
> @README.md 이 프로젝트를 3줄로 요약해줘
```

Claude가 답한다:
```
README.md를 읽었습니다.

당신의 프로젝트는:

1. 고객 피드백을 관리하는 SaaS 플랫폼으로,
   여러 채널(이메일, 슬랙, 웹폼)의 피드백을
   한곳에 모아 분석하는 도구입니다.

2. 주요 사용자는 스타트업의 제품 매니저들이고,
   각 피드백에 대해 자동으로 감정 분석과
   분류를 제공합니다.

3. 현재 알파 단계이며, 초대 사용자 50명 정도가
   테스트 중입니다.

혹시 더 알고 싶은 점이 있으신가요?

>
```

성공이다. Claude와 대화할 준비가 끝났다.

## 실습 1: 간단한 파일 요약

이제 조금 더 복잡한 작업을 해보자.

### 준비
프로젝트 폴더에 다음과 같은 파일들이 있다고 가정:
- `meeting-notes.md`: 어제 회의록
- `user-feedback.csv`: 수집한 유저 피드백 데이터

### 명령
```
> @meeting-notes.md @user-feedback.csv
  회의 내용과 유저 피드백을 함께 분석해서,
  우리가 해야 할 일을 5개 항목으로 정리해줘
```

Claude가 작업한다:
```
두 파일을 분석했습니다.

회의에서 나온 내용:
- Q1 로드맵 확정 필요
- 경쟁사 분석 리포트 필요
- 팀 구성 검토

유저 피드백의 주요 테마:
- 속도 개선 요청 (45%)
- UI 개선 요청 (30%)
- 통합 기능 요청 (25%)

종합하면, 우리가 해야 할 일:

1. [높음] 앱 속도 최적화 - 유저 피드백 1순위
2. [높음] 경쟁사 분석 리포트 완성 - 회의에서 결정
3. [중간] UI/UX 개선 - 피드백 2순위
4. [중간] Q1 로드맵 공식화 - 회의 액션아이템
5. [낮음] 새로운 통합 기능 조사 - 향후 계획

우선순위에 따라 착수 순서를 정하시겠어요?

>
```

### 결과 저장
원하면 결과를 파일로 저장할 수도 있다:
```
> 이 분석을 action-plan.md 파일로 저장해줘

Claude: action-plan.md 파일을 생성했습니다.
파일의 내용은:
# Q1 액션 플랜

## 우선순위 순

### 1. [높음] 앱 속도 최적화
...

파일을 확인하려면: cat action-plan.md

>
```

## 대화 종료 방법

대화를 끝내고 싶을 때:

### 1. `/exit` 명령
```
> /exit

✓ 대화를 종료합니다.
프로젝트 메모가 저장되었습니다 (CLAUDE.md).

$
```

### 2. Ctrl+C 키
```
> ^C

✓ 대화를 종료합니다.

$
```

둘 다 같은 효과다. 이전 대화는 `CLAUDE.md`에 저장되어, 다음에 같은 폴더에서 `claude`를 실행할 때 참고된다.

## 자주 나는 문제 & 해결법

### 문제 1: "claude: command not found"
**원인**: PATH에 등록되지 않았다.

**해결**:
- macOS/Linux:
```
  # 수동으로 PATH 추가
  echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
  source ~/.bashrc
```
- Windows PowerShell: PowerShell을 다시 열어보자.

설치 후 터미널을 재시작하는 게 가장 간단하다.

### 문제 2: "API key invalid"
**원인**: API 키가 만료되거나 잘못됐다.

**해결**:
1. https://platform.claude.com/에서 새 키 생성
2. `claude` 명령 다시 실행
3. 새 키 입력

### 문제 3: "Node.js version too old"
**원인**: Node.js 18 미만.

**해결**:
```
$ node --version
v14.5.0  # 너무 오래됨

# Node.js 업그레이드
brew upgrade node  # macOS
# 또는 nodejs.org에서 새 버전 다운로드
```

### 문제 4: "Network connection failed"
**원인**: 인터넷 연결 문제 또는 Claude API 서버 장애.

**해결**:
1. 인터넷 연결 확인
2. VPN을 사용 중이면, 비활성화 후 시도
3. 몇 분 뒤 다시 시도
