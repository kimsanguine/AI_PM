# 모드와 분석 깊이: 상황별 최적 설정

## 개요
Claude Code는 단순한 AI가 아니라, PM의 작업 스타일에 맞춰 조절할 수 있는 **환경**입니다.
3가지 입력 모드(Edit/Auto-Accept/Plan)와 3단계 분석 깊이(think/think harder/ultrathink)를 조합하면,
상황과 리스크 수준에 맞게 최적화된 워크플로를 구성할 수 있습니다.

---

## 1. 입력 모드: 3가지 선택지

### 1.1 Edit 모드 (기본값): 변경 전 검토

**언제 사용할까:**
- 문서 수정 (PRD, 요구사항)
- 코드 생성 (스크립트, SQL)
- 위험도 높은 작업

**특징:**
- Claude가 제안한 변경사항을 **실행 전** 확인 가능
- Diff 형식으로 추가/삭제/변경 라인 표시
- `y` 또는 `n`으로 수동 승인

**실제 화면 예시:**

```
Terminal 입력:
$ claude code

> PRD의 스코프 섹션을 수정해줘. 마케팅 요청사항 3개를
  우선순위순으로 정렬해서 넣어줘

Claude Code 응답:

📝 다음 변경을 제안합니다:

--- a/docs/prd-v2.md
+++ b/docs/prd-v2.md
@@ -15,6 +15,12 @@ ## Scope

 - 버튼 색상 커스터마이징 (iOS 앱)
 - 어두운 테마 지원

+## Scope Updates (마케팅 요청)
+1. [P0] Slack 공유 기능 추가 (viral coefficient +15%)
+   - 각 대시보드를 Slack 채널에 직접 공유 가능
+   - 미리보기 + 클릭하면 앱으로 이동
+
+2. [P1] 이메일 초대 시스템 개선
+   - 팀 초대 시 팀명 + 역할 미리 표시

Accept changes? [y/n/e(dit)/s(kip)]:
y

✅ 변경 사항이 저장되었습니다.
Modified: docs/prd-v2.md
```

**수동 승인 옵션:**
- `y`: 변경사항 모두 수락
- `n`: 변경사항 거절, 다시 작성 요청
- `e`: 제안된 변경사항을 직접 편집
- `s`: 특정 부분만 스킵

### 1.2 Auto-Accept 모드: 즉시 실행

**언제 사용할까:**
- 반복 작업 (스크린샷 이름 변경 x20)
- 저위험 작업 (데이터 포맷 변환)
- 생산성 우선 상황

**활성화 방법:**
```
Terminal:
$ claude code --auto-accept

또는 실행 중:
Shift+Tab (토글)
```

**실제 화면 예시:**

```
Terminal 입력:
$ claude code --auto-accept

> @user-feedback-raw.csv 이 CSV의 모든 칼럼을
  영어로 번역해줘. 확인 없이 바로 저장해.

Claude Code 응답:

🔄 Auto-Accept 모드 활성화됨
   (변경사항 자동 승인)

📝 번역 중...
- "사용자이름" → "username" ✓
- "피드백내용" → "feedback_text" ✓
- "작성날짜" → "created_at" ✓
...

✅ 완료! 20개 칼럼 번역됨
Modified: user-feedback-raw.csv (2.3KB)
```

**주의:**
- 자동 실행이므로 실수 시 영향 범위가 큼
- 롤백 가능하지만 매번 주의 필요
- 팀과 함께 작업할 때는 Edit 모드 권장

### 1.2.1 --dangerously-skip-permissions: 완전 자율 실행

**무엇인가:**
`--dangerously-skip-permissions`는 Auto-Accept보다 한 단계 더 나아간 플래그입니다.
파일 수정뿐 아니라 **Bash 명령어, 외부 API 호출, 시스템 명령** 등 모든 권한 확인을 건너뜁니다.

**활성화 방법:**
```
Terminal:
$ claude --dangerously-skip-permissions

# 또는 특정 명령과 조합:
$ claude --dangerously-skip-permissions -p "모든 CSV 파일을 분석해서 리포트 생성해줘"
```

**Auto-Accept와의 차이:**

| 구분 | Auto-Accept | --dangerously-skip-permissions |
| --- | --- | --- |
| **파일 읽기/쓰기** | ✅ 자동 승인 | ✅ 자동 승인 |
| **Bash 명령어** | ⚠️ 확인 필요 | ✅ 자동 승인 |
| **외부 API 호출** | ⚠️ 확인 필요 | ✅ 자동 승인 |
| **MCP 도구 사용** | ⚠️ 확인 필요 | ✅ 자동 승인 |
| **npm install 등** | ⚠️ 확인 필요 | ✅ 자동 승인 |

**PM이 사용하는 시나리오:**

```
Terminal 입력:
# 시나리오 1: CI/CD 파이프라인에서 자동 실행
$ claude --dangerously-skip-permissions -p \
  "Linear에서 이번 스프린트 데이터를 가져와서 weekly-report.md를 생성해줘"

# 시나리오 2: 스케줄 태스크에서 무인 실행
# (매주 금요일 자동 브리핑 생성)
$ claude --dangerously-skip-permissions -p \
  "Slack #product 채널의 이번 주 논의를 요약하고 digest.md로 저장해"

# 시나리오 3: 대량 파일 처리 파이프라인
$ claude --dangerously-skip-permissions -p \
  "research/ 폴더의 모든 경쟁사 문서를 읽고 비교표를 만들어줘"
```

**⚠️ PM 판단 포인트: 언제 써야 하는가?**

```
✅ 사용 OK:
- 자동화 스킬/스케줄 태스크 (.claude/skills/)
- CI/CD 파이프라인 내 자동 실행
- 로컬 환경에서 대량 반복 작업
- 읽기 전용 분석 (데이터 수집 → 리포트 생성)

❌ 사용 금지:
- 프로덕션 데이터베이스에 접근하는 작업
- 외부 서비스에 데이터를 보내는 작업 (Slack 메시지 전송 등)
- Git push, 배포 등 되돌리기 어려운 작업
- 처음 실행하는 새로운 워크플로 (먼저 Edit 모드로 테스트)
```

**실전 팁**: 3.4-custom-skills.md의 스킬 자동화와 결합하면 강력합니다. 스킬을 먼저 Edit 모드로 테스트하고, 안정화된 후 `--dangerously-skip-permissions`로 무인(자동) 실행하세요.

### 1.3 Plan 모드: 사전 계획 검토

**언제 사용할까:**
- 복잡한 다단계 작업
- 팀원과 실행 계획 사전 합의
- 큰 변경사항 (대량 리팩토링, 마이그레이션)

**활성화 방법:**
```
Terminal:
$ claude code --plan

또는 중간에:
Shift+Tab → Plan 선택
```

**실제 화면 예시:**

```
Terminal 입력:
$ claude code --plan

> 우리 프로젝트의 사용자 데이터베이스를 PostgreSQL 9.6에서
  14로 업그레이드해야 해. 어떤 단계로 진행할까?

Claude Code 응답:

📋 Plan Mode: 실행 전 계획 검토

🔍 PostgreSQL 9.6 → 14 업그레이드 계획
   (영향도: 높음 | 예상 소요시간: 2-3시간)

Phase 1: 사전 검사 (실제 DB 변경 없음)
├─ 현재 데이터베이스 상태 스캔
├─ 호환성 문제 진단
│  └─ DEPRECATED 함수 찾기 (5개 발견됨)

1. 프로덕션 전 staging에서 먼저 실행
2. 사용자가 적은 시간대 선택 (새벽 2-4시)
3. 팀 슬랙에 일정 공지 + 긴급 연락처 배치

Modify what?

```

**Plan 모드의 이점:**
- 실행 전 전체 맥락 파악
- 팀원과 공유 후 피드백 수집 가능
- 예상 시간, 위험 요소 사전 파악

## 2. 분석 깊이: 상황별 조절

### 2.1 기본 분석: think (기본값)

**사용:**
```
> think about [주제]
```

**특징:**
- 표준 깊이 분석
- 빠른 응답 (평균 3-5초)
- 즉각적 의사결정 필요할 때

**예시:**

```
Terminal 입력:
$ claude code

> think about 이 전환율 데이터

입력 파일 (conversion-funnel.csv):
stage,users,converted,conversion_rate
signup,5000,4500,90%
onboarding,4500,3600,80%
payment_info,3600,2700,75%
checkout,2700,1890,70%
completed,1890,1458,77%

Claude Code 응답 (5초):

📊 기본 분석: 전환 퍼널

단계별 전환율:
1. 가입 → 온보딩: 80% (전형적)
2. 온보딩 → 결제정보: 75% (약간 낮음)
3. 결제정보 → 체크아웃: 70% ⚠️ (병목)
4. 체크아웃 → 완료: 77% (회복)

핵심 발견:
- 전체 초대 → 완료: 29.2% (업계 평균 25%)
- 최대 이탈 지점: 결제정보→체크아웃 (-30%)

빠른 액션:
- 결제 UI 간소화 검토
- 고객 지원팀에 문의 이유 조사 요청
```

### 2.2 심층 분석: think harder

**사용:**
```
> think harder about [주제]
```

**특징:**
- 다각도 검토
- 더 많은 맥락 고려 (경쟁사, 시장, 심리학)
- 응답 시간 ~15초

**예시:**

```
Terminal 입력:
$ claude code

> think harder about 경쟁사 대비 우리 포지셔닝 전략

입력 파일 (@competitive-analysis.md):
# 경쟁사 비교

## Asana
- 가격: $10.99/user/month
- 강점: 엔터프라이즈 기능, SSO
- 약점: 온보딩 복잡, 느린 모바일

## Monday.com
- 가격: $8/user/month
- 강점: 비주얼 보드, 자동화
- 약점: 리포팅 약함, 높은 유지비용

## Our Product (TaskFlow)
- 가격: $6/user/month
- 현재 위치: 가격 경쟁력, 빠른 속도

Claude Code 응답 (15초):

🎯 심층 포지셔닝 분석

1️⃣ 가격 전략 (낮은 진입장벽)
- 현재: $6 (-40% vs Asana, -25% vs Monday)
- 위험: 저가 경쟁으로 마진율 악화 가능
- 기회: SMB 시장 점유 가능성 높음

BUT: 가격만으로는 경쟁 불가능
→ 3개월 내 경쟁 기능 추가 필수

2️⃣ 속도 (차별화 핵심)
- 로딩 시간: 0.8초 vs 경쟁사 평균 2.2초
- 심리학: 0.1초 개선당 만족도 +8%
- 기회: "가장 빠른 PM 도구" 브랜딩 가능

권장: 성과 마케팅에 "속도" 강조
- "초 단위로 결정" (vs "시간 단위 리포팅")
- 성공 사례: 영업팀의 의사결정 시간 60% 감소

3️⃣ 기능 격차 분석
- 우리가 없는 것: SSO (23% 엔터프라이즈가 요구)
- 우리가 잘하는 것: 모바일 온보딩 (2배 빠름)

4️⃣ 고객 심리
- Asana: "안전한 선택" (큰 회사들)
- Monday: "창의적 팀용" (스타트업)
- 우리: "민첩하고 빠른 팀" 포지셔닝 가능

❌ 피해야 할 메시지:
- "Asana보다 저렴" (가격 경쟁은 악순환)
- "모든 기능 복사" (리소스 낭비)

✅ 추천 메시지:
- "시간을 벌어주는 도구" (속도 중심)
- "팀의 선택" (고객 추천 사례)
- "확장 가능한 가격" (SMB → 엔터프라이즈)
```

### 2.3 최대 깊이 분석: ultrathink

**사용:**
```
> ultrathink about [주제]
```

**특징:**
- 전체 전략적 맥락 분석
- Trade-off, 리스크, 장기 영향 고려
- 응답 시간 ~30초 (대신 가장 정교한 답변)
- 큰 결정 전 필수 추천

**예시:**

```
Terminal 입력:
$ claude code

> ultrathink about 내년 제품 로드맵의 우선순위

입력 파일 (@roadmap-candidates.md):
# 2026 로드맵 후보

1. Slack 연동 (개발 4주)
   - 사용자 요청: 높음 (72점)
   - 영향도: 중간 (월 ARR +$15K 예상)

2. API 공개 (개발 8주)
   - 사용자 요청: 낮음 (45점)
   - 영향도: 높음 (파트너 생태계 구축, 불확실성 높음)

3. 마이그레이션 도구 (개발 3주)
   - 사용자 요청: 높음 (68점)
   - 영향도: 낮음 (경쟁사 전환 유도)

4. AI 기반 자동 일정 분배 (개발 12주)
   - 사용자 요청: 매우 높음 (89점)
   - 영향도: 미지수 (기술 리스크 높음)

Claude Code 응답 (30초):

🚀 초장기 전략적 분석: 4개 프로젝트 우선순위

===== ULTRATHINK 모드 =====

## Level 1: 비즈니스 시나리오 분석

### 시나리오 A: "수익성 극대화" 경로
최우선 순위: Slack → 마이그레이션 → API → AI
근거:
- Slack: 즉시 구현 (4주), 확실한 수익 (+15K/월)
- 마이그레이션: 신규 고객 획득 비용 절감
- API: 장기 기반 구축 (하지만 수익 불확실)
- AI: 기술 부채 높음

예상 결과:
- Year 1 추가 ARR: +$180K
- 리스크 낮음 (예측 가능)
- BUT: 차별화 약함 (경쟁사도 가능)

---

### 시나리오 B: "시장 리더십" 경로
최우선 순위: AI → API → Slack → 마이그레이션
근거:
- AI: 경쟁사와의 격차 확대 (12주 개발 = 경쟁사보다 6개월 앞감)
- API: 생태계 장악 (파트너 의존도 증가)
- Slack: 필수 기능 (나중에라도 쉬움)
- 마이그레이션: 이미 경쟁사 맞수

예상 결과:
- 기술 리더 포지셔닝 강화
- BUT: Year 1 수익 불확실 (-$50K ~ +$100K)
- 팀 기술력 성장 (채용/보유 이점)

### 시나리오 C: "균형 수축" 경로 (🎯 권장)
최우선 순위: Slack (4주) → AI 파일럿 (4주 MVP)
→ API (5주) → 마이그레이션 (3주)

근거:
1. Slack: 확실한 수익 (빠른 의욕 고취)
2. AI 파일럿: 기술 검증 + 고객 반응 확인 (리스크 낮춤)
   - 풀 기능 대신 "팀 내 업무 할당 추천" 만 먼저
   - 실제 고객 반응 보고 방향 결정
3. API: 개발자 생태계 기초 (스타트업 세그먼트 Lock-in)
4. 마이그레이션: 성숙기 고객 확보 도구

예상 결과:
- Year 1 ARR: +$160K (수익성 우수)
- 기술 리더십: 중급 (AI 파일럿이 신호 역할)
- 위험 관리: 우수 (기술 및 시장 리스크 분산)

| 선택 | 장점 | 단점 | 제약조건 |
|------|------|------|--------|
| **시나리오 A** | 예측 가능, 빠른 수익 | 차별화 없음, 경쟁사 추격 용이 | 기술팀 모티베이션 ↓ |
| **시나리오 B** | 시장 리더 포지션, 기술력 | 불확실한 수익, 높은 기술 리스크 | IPO/펀딩 필요 |
| **시나리오 C** | 위험 관리 ✓, 균형적 성장 | 소수 고객만 만족 | 병렬 처리 능력 필요 |

시나리오 B는 위험 (시니어 개발자 1명으로 AI 풀 기능 12주는 과부하)

2️⃣ **AI 파일럿 (MVP)** (5-8주)
   - 투자: 낮음 | 리스크: 중간 | 학습: 높음
   - 범위: "팀 내 업무 할당 추천" 기능만
   - 검증: 5개 고객사 파일럿 (프리티어 사용)

3️⃣ **API 공개** (9-13주)
   - 투자: 중간 | 리스크: 낮음 | 생태계: 높음
   - 초기: 스타트업용 무료 Tier (고객 Lock-in)

4️⃣ **마이그레이션 도구** (14-16주)
   - 투자: 낮음 | 리스크: 낮음 | 고객: 경쟁사 이탈
   - 영업 도구로 사용 (신규 영업에 "데이터 가져오기 쉬움" 강조)

## ⚠️ 위험 신호 & 대응

**Risk 1: AI 파일럿이 실패하면?**
→ 최대 손실: 4주 개발
→ 대응: 5개 고객사 피드백 수집 후 중단 결정

**Risk 2: 엔지니어 번아웃?**
→ 신호: 4개 동시 프로젝트는 과부하
→ 대응: Slack (외부 라이브러리) + 마이그레이션 (주니어) 병렬화

**Risk 3: API 수요가 너무 낮으면?**
→ 신호: 파일럿 기간 중 API 요청 < 3개사
→ 대응: 우선순위 down (2027년 미루기)
```

## 3. 모드 조합 가이드: PM 워크플로별 추천 설정

### 워크플로 1: "빠른 의사결정" (마케팅 캠페인 기획)

| 단계 | 모드 | 깊이 | 이유 |
| --- | --- | --- | --- |
| 1단계: 아이디어 생성 | Auto-Accept | think | 빠른 산출 (실패 허용) |
| 2단계: 경쟁사 비교 | Edit | think harder | 시장 맥락 필요 |
| 3단계: 최종 선택 | Plan | think | 전체 요약 검토 |

**예시:**
```
$ claude code --auto-accept

> 이번 분기 마케팅 캠페인 3가지 아이디어 + 각 예산안 빠르게 만들어줘

[3분 안에 3가지 안 제시]

Shift+Tab → Edit 모드로 전환

> 우리 경쟁사 (Asana, Monday.com) 대비 우리 강점을
  think harder로 분석해줘

[각 캠페인별 경쟁사 대비 메시지 포지셔닝]

Shift+Tab → Plan 모드로 전환

> 이 3가지 캠페인의 전체 실행 계획을 보여줘

[공정성 검토 후 최종 선택]
```

---

### 워크플로 2: "신중한 구조 변경" (조직 개편, 대규모 마이그레이션)

| 단계 | 모드 | 깊이 | 이유 |
| --- | --- | --- | --- |
| 1단계: 초안 작성 | Plan | ultrathink | 전체 맥락을 먼저 |
| 2단계: 세부 계획 | Edit | think harder | 구간별 위험 분석 |
| 3단계: 실행 | Auto-Accept | think | 확인된 계획 실행 |

**예시:**
```
$ claude code --plan

> ultrathink about 우리 팀 구조를 Function-based에서
  Product-based로 개편하는 방안

[전체 영향도, 인원 재배치, 리스크 검토]

Shift+Tab → Edit 모드로 전환

> 이 3단계 중 각 단계의 위험 요소를 think harder로 분석해줘

[구간별 상세 위험 분석]

계획 확정 후:

$ claude code --auto-accept

> [실제 변경] 팀 구조도를 이전 파일에서 새로 업데이트해줘

[자동 실행]
```

---

### 워크플로 3: "반복 작업 최적화" (데이터 정제, 문서 포맷 통일)

| 단계 | 모드 | 깊이 | 이유 |
| --- | --- | --- | --- |
| 첫 번째 | Edit | think | 1회차는 검증 |
| 나머지 | Auto-Accept | think | 일관성 유지, 속도 |

**예시:**
```
$ claude code

> 이 20개의 사용자 인터뷰 노트를 "이름_날짜.md" 형식으로
  모두 rename해줄 수 있을까?

[첫 3개 변경 사항 보임]

Accept? y

Shift+Tab → Auto-Accept 모드로 전환

> 나머지 17개도 같은 방식으로 모두 rename해줘

[남은 17개 자동 처리]
```

## 4. 실행 중 제어 명령어

### 4.1 실행 중단

```
Terminal에서 실행 중:
Ctrl+C

Claude가 즉시 중단하고 상태 보고:

 - 파일 3개 수정
 - 파일 1개 생성
완료하시겠어요?"
```

### 4.2 모드 전환

```
실행 중:
Shift+Tab

선택지 표시:
1. Edit 모드 (변경사항 검토)
2. Auto-Accept 모드 (자동 실행)
3. Plan 모드 (계획 재검토)

선택 후 즉시 적용됨
```

### 4.3 깊이 변경

```
> [new task] ultrathink about ...

또는 이전 분석 재평가:
> 위의 think 분석을 ultrathink 레벨로 다시 깊게 분석해줘
```

## 5. 모드 선택 의사결정 트리

```
작업 시작
│
├─ "이건 실수 시 영향이 큰가?"
│  ├─ YES → Edit 모드
│  └─ NO → Auto-Accept 모드
│
├─ "여러 팀원과 합의가 필요한가?"
│  ├─ YES → Plan 모드
│  └─ NO → 위의 선택대로
│
├─ "분석 깊이는?"
│  ├─ 빠른 의사결정 필요 → think
│  ├─ 다각도 검토 필요 → think harder
│  └─ 전략적 의사결정 필요 → ultrathink
```

## 6. 실습 과제

1. 자신의 프로젝트에서 "저위험 반복 작업" 1개 선택
   → Auto-Accept 모드로 5번 실행 (확인 없이)

2. 복잡한 분석 과제 1개 선택
   → 같은 질문을 3가지 깊이(think/harder/ultra)로 실행
   → 결과 차이 비교 (응답 시간 vs 품질)

3. 팀원 2명과 함께 중요한 결정
   → Plan 모드로 미리 계획 검토
   → 피드백 후 실행 (팀 신뢰도 향상)

## 요점 정리

### 입력 모드
| 모드 | 용도 | 실행 | 주의 |
| --- | --- | --- | --- |
| Edit | 검토 필요 (위험도 ↑) | 수동 승인 | 느림 |
| Auto-Accept | 반복/저위험 | 자동 | 롤백 주의 |
| Plan | 다단계/팀 협력 | 사전 검토 | 가장 명확 |

### 분석 깊이
| 깊이 | 응답 시간 | 사용 시점 | 결과 스타일 |
| --- | --- | --- | --- |
| think | ~5초 | 빠른 결정 필요 | 핵심만 |
| think harder | ~15초 | 다각도 검토 | 경쟁 맥락 포함 |
| ultrathink | ~30초 | 전략적 결정 | Trade-off + 리스크 |
