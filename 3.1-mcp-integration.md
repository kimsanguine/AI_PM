# MCP 연동: PM의 도구를 하나로 연결하기

## MCP란 무엇인가

**Model Context Protocol (MCP)**는 Claude Code가 외부 도구(Notion, Linear, Slack, GitHub 등)와 대화하는 표준 프로토콜입니다.

### PM 관점에서의 의미

```
❌ Before: 복붙의 악순환
PRD를 Google Docs에서 작성
→ 링크 복사해서 Slack에 공유
→ 팀원이 읽고 Linear에 티켓 수동 생성
→ 매주 상태 체크하며 Slack에 수동 보고
→ 도구 간 정보 동기화에 하루 1시간 이상 소요

✅ After: 도구 간 대화
Claude Code 터미널: "이 PRD를 Notion에 게시하고,
Linear에 Epic과 Story를 만들어줘"
→ 자동으로 모든 도구가 동기화됨
→ 10분 안에 완료
```

**핵심**: "복붙"에서 "대화"로 전환하기

## PM에게 필요한 4가지 MCP 연결

### 1. Notion MCP — 문서 허브

**Notion의 역할**: PRD, 회의록, 프로젝트 계획 등 모든 문서의 중앙 저장소

#### 설정 방법

```bash
# ~/.claude/mcp_settings.json에 추가
{
  "mcpServers": {
    "notion": {
      "command": "uvx",
      "args": ["mcp-server-notion"],
      "env": {
        "NOTION_API_KEY": "noti_xxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

Notion API Key는 [notion.so/my-integrations](https://notion.so/my-integrations)에서 발급받을 수 있습니다.

#### PM 활용 패턴

```
시나리오: PRD를 작성하고 자동으로 Notion에 게시

Claude Code 터미널:
$ claude
> 지금부터 입력할 PRD를 우리 Notion의
  "프로젝트/Product" 페이지 하위에
  "신규 기능: 추천 시스템" 이라는 제목으로 게시해줄 수 있어?

> [PRD 내용 붙여넣기]

결과:
✓ Notion에 자동 게시됨
✓ 팀원들이 댓글 추가 가능
✓ Claude Code로 댓글 수집 → AI 응답 초안 생성 가능
```

---

### 2. Linear MCP — 티켓 관리

**Linear의 역할**: PRD를 개발 작업 단위(Epic/Story)로 분해

#### 설정 방법

```bash
{
  "mcpServers": {
    "linear": {
      "command": "uvx",
      "args": ["mcp-server-linear"],
      "env": {
        "LINEAR_API_KEY": "lin_api_xxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

Linear API Key는 [linear.app/settings/api](https://linear.app/settings/api)에서 발급받습니다.

#### PM 활용 패턴

```
시나리오: PRD 기반으로 자동 티켓 생성

Claude Code 터미널:
$ claude
> 다음 PRD를 읽고 Linear의 "Q1-신규기능" 프로젝트에
  Epic 1개와 하위 Story 5개를 자동으로 생성해줄래?

> [PRD 붙여넣기]
>
> 조건:
> - Epic 제목: [PRD의 기능명]
> - Story들은 개발 팀이 착수할 수 있는 단위로 분해
> - 각 Story마다 예상 소요 시간(Point) 추정

결과:
✓ Epic 1개 자동 생성
✓ Story 5개 자동 생성 (추정 포인트 포함)
✓ Linear에서 팀이 바로 작업 시작 가능
```

---

### 3. Slack MCP — 커뮤니케이션

**Slack의 역할**: 팀 동기화, 진행 상황 공유, 의사결정 기록

#### 설정 방법

```bash
{
  "mcpServers": {
    "slack": {
      "command": "uvx",
      "args": ["mcp-server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-xxxxxxxxxxxxxxxxxxxxx",
        "SLACK_APP_TOKEN": "xapp-xxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

Slack Bot Token은 [api.slack.com/apps](https://api.slack.com/apps)에서 발급받습니다.

#### PM 활용 패턴

```
시나리오: 주간 상태 보고서 자동 생성 및 공유

Claude Code 터미널:
$ claude
> #weekly-updates 채널에
  이번 주(2월 17-23) 완료 항목, 진행 중 항목, 블로킹 이슈를
  정리해서 보고서 형식으로 올려줄 수 있어?

> Linear 프로젝트 "Q1-신규기능"의 지난 주 상황을
  기반으로 작성해줘.

결과:
✓ Linear 데이터 자동 수집
✓ 보고서 형식으로 정리
✓ #weekly-updates에 자동 포스팅
✓ PM이 손 안 움직임
```

**추가 활용**: Slack 채널에서 의견 수집 후 요약

```
Claude Code:
> #product-decision 채널의 지난 3일간 대화를 읽고
  주요 논점 3개와 합의된 사항을 요약해줄래?

결과:
✓ 채널 대화 자동 수집
✓ AI로 요약 (3분 소요)
✓ PM이 모든 의견을 빠짐없이 확인 가능
```

### 4. GitHub MCP — 코드/배포

**GitHub의 역할**: 개발 진행 상황 실시간 추적

#### 설정 방법

```bash
{
  "mcpServers": {
    "github": {
      "command": "uvx",
      "args": ["mcp-server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_xxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

GitHub Token은 [github.com/settings/tokens](https://github.com/settings/tokens)에서 발급받습니다.

#### PM 활용 패턴

```
시나리오: PR 리뷰 현황 자동 추적

Claude Code:
> repo: {company}/{project}의 지난 1주일 PR 중
  아직 Review 대기 중인 것들을 찾아줄래?

> 각 PR별로:
> - 제목, 작성자
> - 생성된 지 몇 시간 됐는지
> - 파일 변경 수

결과:
✓ 지연된 PR 자동 감지
✓ PM이 개발팀에 검토 요청 가능
✓ 배포 병목 사전 파악
```

## 실전 워크스루: PRD → 티켓 → 상태보고 → 공유

한 번의 Claude Code 세션에서 4개 MCP를 연결하는 실제 시나리오입니다.

### Step 1: PRD 작성 및 Notion 게시

```bash
$ claude

You: "새로운 추천 시스템 PRD를 작성해줄래?
      요구사항:
      - 사용자의 조회 이력 기반 상품 추천
      - 추천율 목표: CTR 5% 상향
      - 론칭: 2026년 4월 말"

Claude Code:
[PRD 자동 생성]

You: "이 PRD를 Notion의 프로젝트 페이지에 게시해줘"

결과: ✓ Notion에 PRD 게시됨
```

### Step 2: PRD 기반 Linear 티켓 생성

```bash
You: "위의 PRD를 읽고 Linear에 구조를 만들어줄래?
      프로젝트: Q1-신규기능
      - 1개의 Epic
      - 5개의 Story (개발 팀이 바로 착수 가능한 단위)
      - 각 Story마다 예상 포인트 추정"

Claude Code:
[Linear 자동 티켓 생성]

결과: ✓ Epic 1개 + Story 5개 생성됨
     ✓ 개발팀이 Linear에서 바로 작업 시작 가능
```

### Step 3: Linear 진행 현황 조회 및 상태 보고서 생성

```bash
You: "Linear 'Q1-신규기능' Epic의 현재 진행 상황을
      읽어서 상태 보고서를 만들어줄래?

      포함할 내용:
      - 완료된 Story: ○○○ (3개)
      - 진행 중 Story: ○○○ (2개)
      - 블로킹 이슈 있으면 명시"

Claude Code:
[Linear에서 데이터 수집]
[상태 보고서 자동 생성]

결과: ✓ 최신 진행 현황 반영된 보고서 작성됨
```

### Step 4: Slack에 공유

```bash
You: "위의 상태 보고서를
      #weekly-updates 채널에 올려줄래?"

Claude Code:
[Slack에 자동 포스팅]

결과: ✓ 전체 팀이 즉시 확인 가능
```

## PM 역할 확장 포인트

### Before: 수동 동기화의 악순환

```
월요일 아침 (9:00-10:00, 1시간 소요)
① 구글 Docs에서 PRD 마지막 검토
② Linear 열어서 팀이 완료한 항목 확인
③ Slack에 주간 목표 공지
④ 지난주 완료 현황을 손으로 정리해서 보고
⑤ GitHub 열어서 배포 상태 확인
⑥ 모든 정보를 정리해서 주간 회의 자료 준비

⚠️ 도구 switching cost: 하루 1시간 이상
⚠️ 정보 중복 입력: PRD → 티켓, 상태 보고서 중복
⚠️ 휴먼 에러: 숫자 빠뜨림, 오래된 정보 공유
```

### After: 자연어 명령으로 전체 흐름 실행

```
월요일 아침 (9:00-9:10, 10분 소요)

Claude Code 터미널:
$ claude
> "지난주 Q1-신규기능 Epic의 진행 현황을 정리해서
   주간 상태 보고서 형식으로 #weekly-updates에 올려줄래?

   그리고 이번주 나와야 할 새로운 PRD도
   Notion 프로젝트 페이지에 게시해줘."

Claude Code가 실행:
✓ Linear에서 지난주 완료 항목 수집
✓ GitHub에서 배포 현황 수집
✓ Slack에서 팀 논의 사항 수집
✓ 모두 합쳐서 상태 보고서 자동 생성
✓ #weekly-updates에 자동 포스팅
✓ Notion에 새 PRD 게시
✓ 완료!

⏱️ 소요 시간: 10분 (PM은 터미널만 모니터링)
⚠️ PM이 판단해야 하는 것:
   - 어떤 정보를 어디에 공유할 것인가?
   - 얼마나 자세히 보고할 것인가?
   - 누구에게 먼저 알릴 것인가?
```

**PM의 진정한 가치**: 도구 조작이 아니라 "판단과 의사결정"

## MCP 연동 시 PM 체크리스트

### 설정 전

```
☐ Notion, Linear, Slack, GitHub 계정 준비됨
☐ 각 서비스별 API Key 발급 확인
☐ ~/.claude/mcp_settings.json 파일 생성 예정
☐ 팀원들에게 연동 사실 공지 (대화 기록 저장되는 것 등)
```

### 설정 후 테스트

```
☐ Claude Code로 Notion 페이지 읽기 테스트
☐ Linear에서 간단한 이슈 생성 테스트
☐ Slack에 테스트 메시지 발송
☐ GitHub 리포 정보 조회 테스트
```

### 일상 운영

```
☐ "Claude Code의 도구 조작"이 팀 권한 정책에 맞는가?
☐ Slack에 게시된 메시지가 적절한 레벨의 정보인가?
☐ Linear의 자동 생성 티켓이 충분히 상세한가?
```

## 실습 과제

### Level 2: MCP 2개 이상 연결 후 실전 체험

**목표**: 본인 프로젝트의 실제 도구 2개 이상을 연결하고,
PRD 작성부터 티켓 생성까지를 한 번에 실행하기

**진행 방식**:

1. **API Key 준비** (10분)
  - Notion / Linear 중 최소 2개 선택
  - API Key 발급
  - ~/.claude/mcp_settings.json에 설정

2. **Claude Code 테스트** (10분)
  - 각 도구에 간단한 명령으로 연결 확인
  - 예: "Notion의 [페이지명] 내용을 읽어줄래?"

3. **실전 PRD 작성** (20분)
  - 본인 팀의 실제 프로젝트 1개 선택
  - Claude Code에서 PRD 작성 요청
  - Notion에 자동 게시

4. **티켓 자동 생성** (15분)
  - PRD를 기반으로 Linear Epic/Story 자동 생성
  - 예상 포인트 추정 포함

5. **결과 검토**
  - Notion: PRD가 제대로 게시됐는가?
  - Linear: 티켓이 개발팀이 이해할 수 있는 단위로 분해됐는가?
  - 소요 시간: 실제로 몇 분 단축됐는가?

**제출 자료** (선택):
- 스크린샷: 생성된 Notion 페이지 + Linear Epic/Story
- 소감문 (1줄): "이전에는 ○○시간 걸렸는데, 이제는 ○○시간만 걸린다"
