# 소크라틱 질문 가이드 — PRD 맥락

> PRD Generator Skill의 Step 2에서 참조하는 질문 라이브러리입니다.
> Richard Paul & Linda Elder의 6가지 유형을 PM 맥락에 맞게 구체화했습니다.

---

## 1. 명확화 질문 (Clarification)

"그 말이 정확히 무슨 뜻인가?"

PM이 모호하게 쓴 용어나 개념을 날카롭게 정의하도록 만듭니다.

| 상황 | 질문 |
| --- | --- |
| "사용자 경험 개선" | "개선"이 구체적으로 어떤 행동 변화를 의미하나요? |
| "온보딩 완료" | "완료"의 기준이 뭔가요? 어떤 액션까지를 완료로 보나요? |
| "핵심 사용자" | 이 기능의 핵심 사용자는 전체 중 몇 %이고, 어떤 특성으로 구분되나요? |
| "빠른 출시" | "빠른"이 2주인가요, 1개월인가요? 무엇을 포기하면 빨라지나요? |
| "확장성 있는 설계" | 어떤 규모(유저 수/데이터량)까지 견뎌야 하나요? |

---

## 2. 가정 탐색 질문 (Assumption Probing)

"당신은 무엇을 전제하고 있는가?"

PM이 무의식적으로 "당연하다"고 생각하는 전제를 드러냅니다.

| 상황 | 질문 |
| --- | --- |
| "이 기능이 필요하다" | 이 기능 없이는 문제가 해결 불가능한가요? 다른 방법은 없나요? |
| "유저가 원할 것이다" | 그 기대가 데이터에 기반한 건가요, 직감에 기반한 건가요? |
| "경쟁사가 했으니 우리도" | 경쟁사의 맥락이 우리와 같은가요? 그들의 결과가 성공적이었나요? |
| "기술적으로 가능하다" | 엔지니어링 팀과 실현 가능성을 확인했나요? 숨겨진 복잡성은? |
| "이번 분기에 가능하다" | 현재 팀의 capacity와 의존성을 고려한 일정인가요? |

---

## 3. 근거 질문 (Evidence)

"그렇게 생각하는 이유는?"

주장의 근거를 요구하여, 데이터 기반 의사결정을 강제합니다.

| 상황 | 질문 |
| --- | --- |
| "이탈률이 높다" | 어떤 기준으로 '높다'고 판단하나요? 업계 벤치마크와 비교하면? |
| "사용자가 불만이다" | 정량적 데이터는 있나요? (NPS, CSAT, 이탈률) 정성적 데이터는? |
| "매출이 늘 것이다" | 어떤 계산 근거로? 유사 기능 출시 시 다른 제품에서의 결과는? |
| "시장이 크다" | TAM/SAM/SOM을 구체적으로 산정했나요? 어떤 소스 기반? |

---

## 4. 관점 전환 질문 (Perspective)

"다른 관점에서 보면 어떤가?"

PM의 시야를 넓혀서 blind spot을 발견합니다.

| 관점 | 질문 |
| --- | --- |
| 엔지니어 | "이 기능의 유지보수 비용은 얼마나 되나요? 기술 부채를 만들지 않나요?" |
| 디자이너 | "이 기능이 기존 UX 흐름을 복잡하게 만들지 않나요?" |
| CS팀 | "이 기능 출시 후 고객 문의가 늘어날 가능성은?" |
| 경영진 | "이 기능이 우리 회사의 연간 전략과 어떻게 정합하나요?" |
| 신규 사용자 | "이 제품을 처음 보는 사람이 이 기능을 이해할 수 있나요?" |
| 경쟁사 | "경쟁사가 이걸 보고 어떻게 반응할까요? 쉽게 따라할 수 있나요?" |

---

## 5. 결과 탐색 질문 (Consequence)

"그렇게 하면 무슨 일이 일어나는가?"

1차 효과 너머의 2차, 3차 효과를 미리 그립니다.

| 상황 | 질문 |
| --- | --- |
| 무료 기능 추가 | "유료 전환 동기가 약해지지 않나요?" |
| 알림 기능 추가 | "알림 피로(notification fatigue)로 이탈이 늘 수 있지 않나요?" |
| AI 자동화 도입 | "사용자가 통제감을 잃는다고 느끼면 어떻게 되나요?" |
| 가격 인상 | "기존 고객의 이탈률 변화는? 매출 증가가 이탈 손실을 상쇄하나요?" |
| 플랫폼 확장 | "유지보수 비용 증가, QA 범위 확대의 비용은 고려했나요?" |

---

## 6. 메타 질문 (Meta-Question)

"왜 이 질문이 중요한가?"

PM이 전체 그림을 보고, 가장 중요한 판단을 식별합니다.

| 질문 |
| --- |
| "이 PRD에서 틀렸을 때 가장 큰 피해를 주는 가정은 무엇인가요?" |
| "6개월 뒤에 이 결정을 후회한다면, 어떤 이유일까요?" |
| "이 기능을 안 만들기로 결정한다면, 어떤 이유가 가장 강력할까요?" |
| "이 PRD를 읽는 사람이 가장 먼저 의심할 부분은 어디일까요?" |
| "지금 가장 불확실한 것 한 가지를 먼저 검증한다면, 뭘 하시겠어요?" |
