# Good 예시 — 신규 사용자 활성화 OST

입력: 가입 후 7일 내 핵심 기능 1회 사용률(activation) 인터뷰 합성 + 사용 로그.

## Outcome (루트)

가입 후 7일 내에 핵심 기능을 처음 사용하는 신규 사용자 비율을 높인다.
(현재 34% → 목표 50%. business 매출이 아니라 사용자 행동 변화로 정의.)

## 트리

- Outcome: 7일 내 핵심 기능 첫 사용률 ↑
  - Opportunity A: 가입 직후 무엇부터 할지 모른다 (빈도 19/30, 63%)
    - Solution A1: 첫 화면에 "3분 안에 끝나는 시작 과업" 1개만 제시
    - Solution A2: 가입 시 목적을 1개 물어 화면을 분기
  - Opportunity B: 첫 시도에서 빈 화면만 보고 가치를 못 느낀다 (빈도 14/30, 47%)
    - Solution B1: 샘플 데이터가 미리 채워진 상태로 시작
    - Solution B2: 동종 팀의 예시 템플릿 갤러리
  - Opportunity C: 이메일 인증 단계에서 이탈한다 (빈도 8/30, 27%)
    - Solution C1: 인증을 첫 사용 이후로 미룸
  - 가설 Opportunity D: 가격이 보여서 망설인다 (2/30, 7%) → 가설로 분리, 트리 우선순위 제외.

## 이번에 집중할 가지

- Opportunity A. 빈도 최고(63%) + Solution 이 저비용. C 는 임팩트 있으나 보안 검토 선행 필요 → 다음 사이클.

## 첫 실험

- 가설: "첫 화면에 시작 과업 1개만 제시하면(A1), 첫 사용까지 걸리는 시간이 줄어 7일 활성화가 오른다."
- 검증 방법: 신규 가입자 50:50 A/B, 2주.
- 성공 기준: 실험군 7일 활성화가 대조군 대비 +8%p 이상.
- 기각 기준: 차이 +2%p 미만 또는 첫 세션 이탈률이 오히려 상승.

## 한계

- Solution A2(목적 분기)는 근거 인터뷰가 A1 보다 약함 — A1 실험 결과 본 뒤 판단.
- Opportunity B 의 두 solution 을 비교(compare-and-contrast)하지 않고 A 가지부터 실험 — B 가지로 넘어갈 때는 B1/B2 동시 비교 필요(Torres 원칙).
- 로그는 데스크톱 위주. 모바일 신규 사용자 행동은 별도 확인 필요.
