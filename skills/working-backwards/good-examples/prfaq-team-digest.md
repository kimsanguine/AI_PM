# Good 예시 — 팀 주간 다이제스트 PR-FAQ

입력: "흩어진 업무 도구의 주간 활동을 한 장으로 요약" 아이디어. 근거 = 팀 리드 12명 인터뷰.

## Press Release

**헤드라인**: 팀 리드가 월요일 아침 5분 만에 지난주를 파악한다

**서브헤드**: 여러 도구에 흩어진 활동을 매주 월요일 한 페이지 요약으로 자동 정리.

**문제**: 팀 리드는 매주 여러 도구를 돌며 지난주 무슨 일이 있었는지 직접 모은다. 인터뷰 12명 중 9명이 "월요일 오전 1시간 이상을 현황 취합에 쓴다" 고 답했다.

**해결**: 주간 다이제스트는 연결된 도구의 지난 7일 활동을 한 페이지로 모은다. 무엇이 끝났고, 무엇이 막혔고, 누가 무엇을 기다리는지 한눈에 보인다.

**가상 고객 인용**:
> "예전엔 월요일 오전을 통째로 현황 모으는 데 썼는데, 이제 커피 마시며 한 장만 읽으면 회의 준비가 끝난다." — 30인 팀 리드

**시작하는 법**: 도구를 연결하고 월요일 받을 시각만 정하면 끝.

## FAQ — 고객용

- **기존 대시보드와 뭐가 다른가?** 대시보드는 실시간 지표, 이것은 "지난주 서사" 다. 무엇이 변했는지를 문장으로 준다.
- **데이터는 안전한가?** 연결된 도구의 읽기 권한만 사용하고, 원문은 저장하지 않는다. [가정 — 보안 설계 검토 전]
- **요약이 틀리면?** 각 항목에 원문 링크가 있어 한 번에 검증 가능.

## FAQ — 내부용

- **왜 우리가? 도구 벤더가 직접 하면 끝 아닌가?** 단일 벤더는 자기 도구만 본다. 우리의 가치는 도구 횡단 통합 — 단, 벤더가 통합 API 를 닫으면 위협. [가정]
- **왜 지금?** 인터뷰 9/12 가 같은 페인 → 타이밍 근거. 다만 segment 가 팀 리드에 한정.
- **실패 조건은?** 요약 신뢰도가 낮으면 "또 직접 확인" 으로 회귀 → 가치 0.
- **안 만들면?** 리드들은 지금처럼 수동 취합 유지. 치명적이지 않음 → 긴급도는 중간.

## 검증 신호

- 통과: 프로토타입 다이제스트를 받은 리드의 ≥ 60% 가 "월요일 취합 시간이 줄었다" 고 응답.
- 기각: 요약 신뢰 부족으로 절반 이상이 원문을 다시 전수 확인하면 만들지 않는다.
