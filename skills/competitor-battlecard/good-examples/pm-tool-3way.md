# Good 예시 — PM 도구 3사 비교

## 비교 축
| 축 | 우리 | A (Linear-like) | B (Notion-AI-like) | C (custom GPT) |
|---|---|---|---|---|
| Target | mid-market PM (50–500 emp) | startup eng team | knowledge worker general | individual maker |
| Core | Claude Code 기반 워크플로우 | 이슈 트래킹 + AI 요약 | 문서 + AI Q&A | 단발 프롬프트 도구 |
| Price | $29/mo | $19/seat/mo | $20/mo | $20/mo |
| Public users (2026) | early | ~50k teams | ~10M | 추정 1M+ |
| Differentiator | 컨텍스트 인프라 (CLAUDE.md/INDEX.md/Skills) | 통합 워크플로우 | 콘텐츠 검색 | 진입 장벽 낮음 |
| Weakness | 신규, 학습 곡선 | PM 워크플로우 깊이 부족 | 워크플로우보다 검색 | 폐루프 없음 |

## 배틀카드 — A
- **그들의 강점**: 통합된 이슈 트래킹 + AI 요약, eng 팀 친화적 UI.
- **그들의 약점**: PRD 깊이 부족, 소크라틱 PRD·전략 도구 없음.
- **우리의 differentiator**: PM 의 컨텍스트 인프라(CLAUDE.md/INDEX.md/Skills) 자체를 가르침.
- **그들의 공격**: "우리는 이슈와 AI 가 통합돼 있는데, 너는 도구를 또 도입해야 하지?"
- **우리의 응답**: "우리는 도구가 아니라 **운영 방식** 이다. 이슈 트래커는 그대로 쓰되, 그 위에 결정의 폐루프를 얹는 가이드다."

## 배틀카드 — B
- **그들의 강점**: 거대 도큐먼트 베이스 + 검색.
- **그들의 약점**: 워크플로우 자동화 부재, PRD 템플릿이 일반적.
- **우리의 differentiator**: 8축 CLAUDE.md / 검증 사다리 같은 **PM 전용** 프레임워크.
- **그들의 공격**: "우리는 이미 모든 회사가 쓴다."
- **우리의 응답**: "도큐먼트는 그쪽에 두라. 우리는 그 위에 PM 의 결정 회로를 만든다."
