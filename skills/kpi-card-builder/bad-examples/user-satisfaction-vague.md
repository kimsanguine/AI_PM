# Bad 예시 — "사용자 만족도" 카드 (작성 거부했어야 함)

## 무엇이 문제인가
- "사용자 만족도" 가 측정 가능 형태로 분해되지 않음
- 계산식이 ambiguous ("점수 평균")
- Source 가 "내부 도구" — 어느 도구, 어느 테이블 불명
- 임계치가 "좋음 / 보통 / 나쁨" — 숫자 없음
- 반反사례 절 부재
- 마법 단어 ("만족도") 가 그대로 KPI 이름

## 출력 (이렇게 하지 말 것)

## KPI Card — 사용자 만족도

### 정의
사용자가 우리 제품에 만족하는 정도.

### 계산식
점수 평균.

### Source
- 내부 도구

### 임계치
| Level | 행동 |
|---|---|
| 좋음 | 그대로 둔다 |
| 보통 | 개선 검토 |
| 나쁨 | 조치 |

## 왜 나쁜가
- "만족도" → 분해되지 않은 마법 단어. 분해해야 함.
- 계산식이 SQL/수식이 아님. 검증 불가.
- Source 가 추상적. 재현 불가.
- 임계치 숫자 없음. action trigger 가 안 됨.
- 반反사례 없음. 이 KPI 가 무엇을 못 잡는지 모름.

## 올바른 응답 (작성 거부 + 재정의 제안)

이 KPI 는 측정 가능 형태로 분해해야 작성할 수 있습니다. 후보 3개:

1. **NPS** — 0–10 추천 의향. 표본 임계치 N≥100/월.
2. **CSAT** — 슬래시 사용 직후 1–5 평가. 응답률 ≥ 20%.
3. **재방문률** — D7 retention, D30 retention.

어느 것이 의사결정에 쓰기 가장 좋은지 정하시면 그 KPI 의 카드를 작성합니다.
