# 2.7 Hooks — Claude Code 폐루프 자동화

> Hooks 는 Claude Code 세션 안에 **사용자가 정의한 셸 스크립트를 끼워 넣는 지점** 이다. 모델에게 매번 "이걸 해줘" 라고 말하는 대신, 시스템 차원에서 자동으로 일어나게 한다. 챕터 2.6 에서 본 검증 사다리의 1단(자동 포맷) 이 실제로 매 편집마다 강제되는 방식이 바로 이 hooks 다.

---

## 무엇이 바뀌는가

| 도구 명령 없이 매번 사람이 하던 것 | Hook 으로 자동화되는 것 |
|---|---|
| 편집 후 "린터 돌려줘" 한 번 더 입력 | `PostToolUse` 가 자동 실행 |
| 세션 시작 시 INDEX.md 를 다시 알려줌 | `SessionStart` 가 컨텍스트로 주입 |
| 병렬 세션 중 어느 게 끝났는지 모름 | `Stop` 이 소리·알림으로 통보 |

Hook 의 본질은 "**룰을 매 세션 사람이 말로 강제하지 않는다**" 다. 룰은 시스템에 박는 것.

## 5가지 Hook 종류

Claude Code 가 제공하는 hook 이벤트 (2026-05 기준):

| 이벤트 | 언제 발화 | 흔한 용도 |
|---|---|---|
| `SessionStart` | 세션 시작 직후 | INDEX.md / 환경 정보 컨텍스트 주입 |
| `UserPromptSubmit` | 사용자가 프롬프트 제출 직전 | 프롬프트 변환, prefix 자동 추가 |
| `PreToolUse` | 도구 실행 직전 | 보안 차단(특정 명령 금지), 사용자 동의 |
| `PostToolUse` | 도구 실행 직후 | 자동 포매터, 파일 변경 로깅 |
| `Stop` | 세션 종료 시 | 알림, 트랜스크립트 보관, 정리 |

## 설치 위치

```
~/.claude/
├── settings.json          # hook 등록 위치
└── hooks/
    ├── stop-notify.sh
    ├── posttooluse-fmt.sh
    └── sessionstart-load.sh
```

본 레포의 `templates/hooks/` 에 3종 스타터가 있다 — 그대로 복사하면 시작점이 된다.

## settings.json 등록 예

```json
{
  "hooks": {
    "SessionStart": [
      { "command": "~/.claude/hooks/sessionstart-load.sh" }
    ],
    "PostToolUse": [
      {
        "matcher": { "tool_name": ["Edit", "Write"] },
        "command": "~/.claude/hooks/posttooluse-fmt.sh"
      }
    ],
    "Stop": [
      { "command": "~/.claude/hooks/stop-notify.sh" }
    ]
  }
}
```

`matcher` 는 옵션이고, 없으면 모든 도구 호출에 발화한다.

## Hook 3종 워크스루

### 1) SessionStart — INDEX.md 주입

```bash
#!/usr/bin/env bash
echo "=== Repo entrypoint =========="
[[ -f INDEX.md ]] && head -80 INDEX.md
[[ -f CLAUDE.md ]] && head -40 CLAUDE.md
```

세션 시작마다 "이 레포가 무엇이고 무엇이 어디에 있는지" 를 모델에 자동 주입한다. 매번 "INDEX.md 먼저 봐줘" 라고 말할 필요 없음.

### 2) PostToolUse — 편집 직후 포매터

```bash
#!/usr/bin/env bash
FILE_PATH=$(jq -r '.tool_input.file_path // empty')
case "$FILE_PATH" in
  *.py)  ruff format "$FILE_PATH" && ruff check --fix "$FILE_PATH" ;;
  *.md|*.json) prettier --write "$FILE_PATH" ;;
esac
```

검증 사다리 1단을 **모든 편집마다** 강제. PM 이 "이 PR lint 통과했어?" 라고 다시 묻지 않아도 됨.

### 3) Stop — 세션 종료 알림

```bash
#!/usr/bin/env bash
LABEL=$(basename "$PWD")
if [[ "$(uname)" == "Darwin" ]]; then
  afplay /System/Library/Sounds/Glass.aiff
  osascript -e "display notification \"세션 완료: ${LABEL}\" with title \"Claude Code\""
fi
```

병렬 세션 3–6개를 돌릴 때 핵심. 어느 세션이 끝났는지 즉시 알 수 있다.

## PM 관점에서 hook 의 가치

- **반복 지시 제거** — "lint 돌려줘", "INDEX.md 봐줘" 같은 매 세션 반복을 제로로.
- **검증 사다리 강제** — 자동 포맷 단계를 모델 의지에 맡기지 않는다.
- **병렬 운영 가시성** — 화면 안 보고 있어도 끝난 세션을 안다.
- **트랜스크립트 폐루프** — `/retro` (챕터 3.3 의 슬래시) 와 결합해, 어디가 자주 실패하는지 → CLAUDE.md / 스킬로 흡수.

## 안티패턴

- Hook 안에서 LLM 호출 또는 long-running 작업 — 세션 latency 망가뜨림. hook 은 ms~수 초.
- 같은 룰을 hook 과 CLAUDE.md 양쪽에 박기 — One rule = one place 위반.
- `PreToolUse` 에서 무거운 보안 검증 — 도구 실행 자체를 느리게 만든다. 별도 백그라운드 검사로.
- 실패 시 exit 1 폭주 — hook 은 기본적으로 `|| true` 로 묶어 세션이 죽지 않게 한다.

## 검증 사다리와 hook 의 매핑

| 사다리 단계 | 어디서 일어나는가 |
|---|---|
| 1단 자동 포맷 | `PostToolUse` (편집 직후) |
| 2단 테스트·링크 | CI 워크플로우 (`.github/workflows/`) — hook 아님 |
| 3단 시각 렌더 | 사람 (또는 Claude Code on the web — 챕터 3.6 예정) |
| 4단 LLM review | 별도 세션 + `engineering-reviewer` 서브에이전트 |

→ **사다리 1단만이 hook 의 영역**. 나머지는 CI 와 사람·서브에이전트의 영역이다. hook 의 범위를 작게 유지하는 것이 핵심.

## 예제 — 본 레포의 hook 운영

본 가이드 자체가 적용하는 hook (`templates/hooks/`):

1. `sessionstart-load.sh` — 세션 시작 시 INDEX.md head 80줄 + CLAUDE.md head 40줄 주입.
2. `posttooluse-fmt.sh` — `.md` 편집 후 prettier (선택), `.py` 편집 후 ruff.
3. `stop-notify.sh` — 종료 시 시스템 알림.

이 3개를 `~/.claude/hooks/` 로 복사한 직후, settings.json 에 위 JSON 등록만 하면 폐루프가 돌기 시작한다.

## 마무리

Hook 은 "모델에게 매번 시키지 말고 시스템에 박아둬라" 라는 룰의 도구화다. 룰을 룰로 두는 한 PM 은 같은 지시를 평생 반복하게 된다. Hook 으로 옮기면, 룰이 코드가 된다 — 그리고 코드는 잊지 않는다.

다음 챕터 2.6(HITL) 의 검증 사다리, 챕터 3.4 의 스킬, 챕터 3.5 의 외부 자동화(n8n)는 모두 hook 과 보완 관계다. Hook 이 작은 폐루프 (편집 후 즉시) 라면, n8n 은 큰 폐루프 (Notion ↔ GitHub promotion) 다.
