# 폐루프 운영 — `/retro` 주 1회

> AI 와의 협업은 한 번 세팅하고 끝이 아니라, 매주 트랜스크립트를 돌아보고 자주 고치는 것을 CLAUDE.md / 스킬로 흡수하는 **폐루프**다. 이 문서가 그 운영 가이드.

---

## 1. 왜 폐루프인가

- AI 는 매 세션 백지에서 시작한다. 사용자가 "그게 아니라", "또 까먹었네" 라고 반복하는 항목이 있다면, 그 룰을 **사용자가 매번 말하는 것**이 아니라 **CLAUDE.md / 스킬에 박아두는 것**이 정답이다.
- 폐루프 없이는 같은 지시를 평생 반복한다.

## 2. 주 1회 워크플로우

### Step 1. 트랜스크립트 수집
- `~/.claude/projects/<hash>/*.jsonl` (Claude Code 세션 로그)
- 최근 7일 / 약 300~2000 턴 범위

### Step 2. 시그널 패턴 grep
다음 표현을 grep:

| 패턴 | 의미 |
|---|---|
| "그게 아니라", "다시 해" | 모델이 의도와 다르게 동작 → 룰 명시 부족 |
| "또 까먹었네", "아직도 틀림" | 같은 실수 반복 → CLAUDE.md 누락 |
| "(왜) X 안 했어?" | 묵시적 기대 → 명시 필요 |
| "출력을 X 형식으로" (3회+) | 포맷 룰 누락 → 4섹션의 Format 미정의 |
| "잘했어" / "완벽" | 좋은 패턴 → 스킬 예시로 흡수 |

### Step 3. 분류 결정 표

| 패턴 종류 | 흡수처 |
|---|---|
| 톤·언어 일반 | `~/.claude/CLAUDE.md` (전역) |
| 본 프로젝트 규칙 | 레포 `CLAUDE.md` (로컬) |
| 반복 워크플로우 | `skills/<name>/SKILL.md` 또는 `templates/commands/<name>.md` |
| 한 번뿐인 지시 | **어디에도 박지 않는다** (스팸 방지) |

### Step 4. PR 1건으로 흡수
- 제목: `chore(retro): WW 2026 absorb N rules`
- 본문: 어떤 트랜스크립트 시그널을 어디에 흡수했는지 표.

## 3. `/retro` 슬래시 (예정 자산)

본 가이드는 `/retro` 커맨드를 제공해 위 흐름을 자동화한다. 사용 예:

```
/retro --since 2026-05-12
```

출력:
- 최근 트랜스크립트 시그널 표 (패턴 / 빈도 / 예시 인용)
- 분류 결정안 (어디로 흡수할지)
- PR 초안 (해당 파일에 어떤 줄을 추가할지)

## 4. One rule = one place

- 같은 룰이 전역 `~/.claude/CLAUDE.md` 와 레포 `CLAUDE.md` 양쪽에 있으면 **한쪽 삭제**.
- 같은 워크플로우가 슬래시와 스킬 양쪽에 있으면 **스킬을 정본**, 슬래시는 스킬 호출 한 줄.
- 중복은 미래의 충돌이다.

## 5. 분기 점검 — 룰 정원 관리

- 분기 1회: CLAUDE.md / skills / commands 전체를 훑고
  - 더 이상 쓰지 않는 룰 → 삭제
  - 중복 룰 → 합치기
  - 작성 후 6개월 손 안 댄 스킬 → 삭제 후보
- 룰은 늘리는 것이 아니라 **줄이는 것이 목표**. 작아야 모델이 따른다.
