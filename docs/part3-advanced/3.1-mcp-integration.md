# MCP 연동: PM의 도구를 하나로 연결하기

## MCP란 무엇인가

**Model Context Protocol (MCP)**는 Claude Code가 외부 도구(Notion, Linear, Slack, GitHub 등)와 대화하는 표준 프로토콜입니다.

> **설정은 터미널 한 줄로 끝납니다.** 예전에는 설정 파일(JSON)을 직접 손으로 편집해야 했지만, 지금은 `claude mcp add …` 명령 한 줄이면 됩니다. JSON을 열어 키 이름을 맞출 필요가 없습니다. 비개발자 PM도 명령어를 복사해서 도구 이름과 주소만 바꾸면 연결됩니다.

### PM 관점에서의 의미

```
❌ Before: 복붙의 악순환
PRD를 Google Docs에서 작성
→ 링크 복사해서 Slack에 공유
→ 팀원이 읽고 Linear에 티켓 수동 생성
→ 매주 상태 체크하며 Slack에 수동 보고
→ 도구 간 정보 동기화에 하루 1시간 이상 소요

✅ After: 도구 간 대화
Claude Code 터미널: "이 PRD를 Notion에 게시하고,
Linear에 Epic과 Story를 만들어줘"
→ 자동으로 모든 도구가 동기화됨
→ 10분 안에 완료
```

**핵심**: "복붙"에서 "대화"로 전환하기

---

## PM에게 필요한 4가지 MCP 연결

### 1. Notion MCP — 문서 허브

**Notion의 역할**: PRD, 회의록, 프로젝트 계획 등 모든 문서의 중앙 저장소

#### 설정 방법

Notion은 Notion이 직접 운영하는 **원격(hosted) MCP 서버**입니다. 내 컴퓨터에 무언가를 설치할 필요 없이, 터미널에 한 줄만 입력하면 됩니다.

```bash
claude mcp add --transport http notion https://mcp.notion.com/mcp
```

연결 후 Claude Code 안에서 `/mcp` 를 입력하면 브라우저가 열립니다. 거기서 Notion 계정으로 로그인(OAuth)하면 인증이 끝납니다. API Key를 직접 발급하거나 복사할 필요가 없습니다.

> **원격 MCP가 무슨 뜻인가요?** Notion·Linear·GitHub 같은 주요 SaaS 도구는 회사가 직접 MCP 서버를 운영합니다. 그래서 내 컴퓨터에서 프로그램을 돌리는 게 아니라, 인터넷 주소(`https://…/mcp`)로 연결하고 로그인으로 인증합니다. 이 로그인 방식을 **OAuth 커넥터**라고 부릅니다.

#### PM 활용 패턴

```
시나리오: PRD를 작성하고 자동으로 Notion에 게시

Claude Code 터미널:
$ claude
> 지금부터 입력할 PRD를 우리 Notion의
  "프로젝트/Product" 페이지 하위에
  "신규 기능: 추천 시스템" 이라는 제목으로 게시해줄 수 있어?

> [PRD 내용 붙여넣기]

결과:
✓ Notion에 자동 게시됨
✓ 팀원들이 댓글 추가 가능
✓ Claude Code로 댓글 수집 → AI 응답 초안 생성 가능
```

---

### 2. Linear MCP — 티켓 관리

**Linear의 역할**: PRD를 개발 작업 단위(Epic/Story)로 분해

#### 설정 방법

Linear도 원격(hosted) MCP 서버입니다. 터미널 한 줄로 연결합니다.

```bash
claude mcp add --transport http linear https://mcp.linear.app/mcp
```

연결 후 `/mcp` 를 입력해 브라우저에서 Linear 계정으로 로그인(OAuth)하면 됩니다. 따로 API Key를 발급할 필요가 없습니다.

#### PM 활용 패턴

```
시나리오: PRD 기반으로 자동 티켓 생성

Claude Code 터미널:
$ claude
> 다음 PRD를 읽고 Linear의 "Q1-신규기능" 프로젝트에
  Epic 1개와 하위 Story 5개를 자동으로 생성해줄래?

> [PRD 붙여넣기]
>
> 조건:
> - Epic 제목: [PRD의 기능명]
> - Story들은 개발 팀이 착수할 수 있는 단위로 분해
> - 각 Story마다 예상 소요 시간(Point) 추정

결과:
✓ Epic 1개 자동 생성
✓ Story 5개 자동 생성 (추정 포인트 포함)
✓ Linear에서 팀이 바로 작업 시작 가능
```

---

### 3. Slack MCP — 커뮤니케이션

**Slack의 역할**: 팀 동기화, 진행 상황 공유, 의사결정 기록

#### 설정 방법

Slack도 원격(hosted) MCP 서버로 연결합니다.

```bash
claude mcp add --transport http slack https://mcp.slack.com/mcp
```

연결 후 `/mcp` 를 입력해 브라우저에서 Slack 워크스페이스에 로그인(OAuth)하면 됩니다. 사용 가능한 도구가 어떤 회사에서 제공하는지·어떤 권한을 요구하는지는 [Anthropic 커넥터 디렉터리](https://claude.ai/directory)에서 확인할 수 있습니다.

#### PM 활용 패턴

```
시나리오: 주간 상태 보고서 자동 생성 및 공유

Claude Code 터미널:
$ claude
> #weekly-updates 채널에
  이번 주(2월 17-23) 완료 항목, 진행 중 항목, 블로킹 이슈를
  정리해서 보고서 형식으로 올려줄 수 있어?

> Linear 프로젝트 "Q1-신규기능"의 지난 주 상황을
  기반으로 작성해줘.

결과:
✓ Linear 데이터 자동 수집
✓ 보고서 형식으로 정리
✓ #weekly-updates에 자동 포스팅
✓ PM이 손 안 움직임
```

**추가 활용**: Slack 채널에서 의견 수집 후 요약

```
Claude Code:
> #product-decision 채널의 지난 3일간 대화를 읽고
  주요 논점 3개와 합의된 사항을 요약해줄래?

결과:
✓ 채널 대화 자동 수집
✓ AI로 요약 (3분 소요)
✓ PM이 모든 의견을 빠짐없이 확인 가능
```

### 4. GitHub MCP — 코드/배포

**GitHub의 역할**: 개발 진행 상황 실시간 추적

#### 설정 방법

GitHub도 원격(hosted) MCP 서버입니다. 다만 로그인(OAuth) 대신 **토큰을 헤더로 전달**하는 방식을 씁니다. [GitHub 토큰 설정](https://github.com/settings/personal-access-tokens)에서 Claude가 다룰 저장소에 접근 가능한 토큰을 발급한 뒤, 아래 한 줄로 연결합니다.

```bash
claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \
  --header "Authorization: Bearer 발급받은_토큰"
```

`--header` 는 토큰을 안전하게 전달하는 부분입니다. `발급받은_토큰` 자리에 본인 토큰을 적어 넣으면 됩니다.

#### PM 활용 패턴

```
시나리오: PR 리뷰 현황 자동 추적

Claude Code:
> repo: {company}/{project}의 지난 1주일 PR 중
  아직 Review 대기 중인 것들을 찾아줄래?

> 각 PR별로:
> - 제목, 작성자
> - 생성된 지 몇 시간 됐는지
> - 파일 변경 수

결과:
✓ 지연된 PR 자동 감지
✓ PM이 개발팀에 검토 요청 가능
✓ 배포 병목 사전 파악
```

## MCP 설정 통합 예시

### 4가지 도구를 한 번에 연결하기

JSON 파일을 손으로 편집하지 않습니다. 터미널에 아래 네 줄을 차례로 입력하면 끝입니다.

```bash
claude mcp add --transport http notion https://mcp.notion.com/mcp
claude mcp add --transport http linear https://mcp.linear.app/mcp
claude mcp add --transport http slack  https://mcp.slack.com/mcp
claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \
  --header "Authorization: Bearer 발급받은_토큰"
```

입력 후 Claude Code에서 `/mcp` 를 실행하면, 로그인(OAuth)이 필요한 서버는 브라우저로 인증을 안내합니다. 현재 연결된 도구 목록은 `claude mcp list` 로 확인할 수 있습니다.

### 어디에 저장될까? — 범위(scope) 고르기

`claude mcp add` 에 `--scope` 를 붙이면 이 연결이 **어디까지 적용될지**를 정합니다. 비개발자 PM이 기억할 것은 세 가지뿐입니다.

| `--scope` 값 | 적용 범위 | 저장 위치 |
| --- | --- | --- |
| `local` (기본값) | 지금 이 프로젝트, 나만 | `~/.claude.json` |
| `project` | 이 프로젝트, 팀 전체 공유 | 프로젝트 폴더의 `.mcp.json` |
| `user` | 내 모든 프로젝트 | `~/.claude.json` |

```bash
# 팀과 공유하고 싶을 때 (프로젝트 폴더에 .mcp.json 생성됨 → Git으로 공유)
claude mcp add --transport http --scope project notion https://mcp.notion.com/mcp
```

`--scope project` 로 추가하면 프로젝트 폴더에 `.mcp.json` 파일이 자동으로 만들어집니다. 이 파일을 Git에 올리면 팀원 전체가 같은 도구 연결을 그대로 받게 됩니다.

> **JSON으로 직접 추가하고 싶다면?** 도구 문서에 JSON 형태의 설정이 주어졌을 때는 `claude mcp add-json <이름> '<json>'` 으로 한 번에 추가할 수 있습니다. 평소에는 위의 `claude mcp add` 한 줄이면 충분합니다.
>
> **로컬에서 직접 실행하는 도구라면?** 일부 도구(예: 사내 스크립트, 로컬 데이터베이스)는 내 컴퓨터에서 프로그램을 직접 돌립니다. 이때는 `--env KEY=값` 으로 API Key 같은 환경 변수를 전달합니다. 예) `claude mcp add --env AIRTABLE_API_KEY=내키 airtable -- npx -y airtable-mcp-server`. Notion·Linear·GitHub처럼 회사가 운영하는 주요 도구는 대부분 위의 원격 방식이라 이 과정이 필요 없습니다.

## 실전 워크스루: PRD → 티켓 → 상태보고 → 공유

한 번의 Claude Code 세션에서 4개 MCP를 연결하는 실제 시나리오입니다.

### Step 1: PRD 작성 및 Notion 게시

```bash
$ claude

You: "새로운 추천 시스템 PRD를 작성해줄래?
      요구사항:
      - 사용자의 조회 이력 기반 상품 추천
      - 추천율 목표: CTR 5% 상향
      - 론칭: 2026년 4월 말"

Claude Code:
[PRD 자동 생성]

You: "이 PRD를 Notion의 프로젝트 페이지에 게시해줘"

결과: ✓ Notion에 PRD 게시됨
```

### Step 2: PRD 기반 Linear 티켓 생성

```bash
You: "위의 PRD를 읽고 Linear에 구조를 만들어줄래?
      프로젝트: Q1-신규기능
      - 1개의 Epic
      - 5개의 Story (개발 팀이 바로 착수 가능한 단위)
      - 각 Story마다 예상 포인트 추정"

Claude Code:
[Linear 자동 티켓 생성]

결과: ✓ Epic 1개 + Story 5개 생성됨
     ✓ 개발팀이 Linear에서 바로 작업 시작 가능
```

### Step 3: Linear 진행 현황 조회 및 상태 보고서 생성

```bash
You: "Linear 'Q1-신규기능' Epic의 현재 진행 상황을
      읽어서 상태 보고서를 만들어줄래?

      포함할 내용:
      - 완료된 Story: ○○○ (3개)
      - 진행 중 Story: ○○○ (2개)
      - 블로킹 이슈 있으면 명시"

Claude Code:
[Linear에서 데이터 수집]
[상태 보고서 자동 생성]

결과: ✓ 최신 진행 현황 반영된 보고서 작성됨
```

### Step 4: Slack에 공유

```bash
You: "위의 상태 보고서를
      #weekly-updates 채널에 올려줄래?"

Claude Code:
[Slack에 자동 포스팅]

결과: ✓ 전체 팀이 즉시 확인 가능
```

## PM 역할 확장 포인트

### Before: 수동 동기화의 악순환

```
월요일 아침 (9:00-10:00, 1시간 소요)
① 구글 Docs에서 PRD 마지막 검토
② Linear 열어서 팀이 완료한 항목 확인
③ Slack에 주간 목표 공지
④ 지난주 완료 현황을 손으로 정리해서 보고
⑤ GitHub 열어서 배포 상태 확인
⑥ 모든 정보를 정리해서 주간 회의 자료 준비

⚠️ 도구 switching cost: 하루 1시간 이상
⚠️ 정보 중복 입력: PRD → 티켓, 상태 보고서 중복
⚠️ 휴먼 에러: 숫자 빠뜨림, 오래된 정보 공유
```

### After: 자연어 명령으로 전체 흐름 실행

```
월요일 아침 (9:00-9:10, 10분 소요)

Claude Code 터미널:
$ claude
> "지난주 Q1-신규기능 Epic의 진행 현황을 정리해서
   주간 상태 보고서 형식으로 #weekly-updates에 올려줄래?

   그리고 이번주 나와야 할 새로운 PRD도
   Notion 프로젝트 페이지에 게시해줘."

Claude Code가 실행:
✓ Linear에서 지난주 완료 항목 수집
✓ GitHub에서 배포 현황 수집
✓ Slack에서 팀 논의 사항 수집
✓ 모두 합쳐서 상태 보고서 자동 생성
✓ #weekly-updates에 자동 포스팅
✓ Notion에 새 PRD 게시
✓ 완료!

⏱️ 소요 시간: 10분 (PM은 터미널만 모니터링)
⚠️ PM이 판단해야 하는 것:
   - 어떤 정보를 어디에 공유할 것인가?
   - 얼마나 자세히 보고할 것인가?
   - 누구에게 먼저 알릴 것인가?
```

**PM의 진정한 가치**: 도구 조작이 아니라 "판단과 의사결정"

## MCP 연동 시 PM 체크리스트

### 설정 전

```
☐ Notion, Linear, Slack, GitHub 계정 준비됨
☐ 원격 도구는 로그인(OAuth)으로 인증 → 별도 발급 불필요
☐ GitHub처럼 토큰이 필요한 도구만 미리 발급
☐ `claude mcp add …` 한 줄로 연결할 준비
☐ 팀원들에게 연동 사실 공지 (대화 기록 저장되는 것 등)
```

### 설정 후 테스트

```
☐ Claude Code로 Notion 페이지 읽기 테스트
☐ Linear에서 간단한 이슈 생성 테스트
☐ Slack에 테스트 메시지 발송
☐ GitHub 리포 정보 조회 테스트
```

### 연동 확인 체크리스트

설치 직후 다음 순서로 확인하세요.

1. `claude mcp list` — 등록된 서버 목록에 표시되는지 확인
2. 실제 데이터 요청으로 테스트:
   - Notion: "최근 페이지 1개 제목을 가져와봐"
   - Linear: "현재 할 일 목록을 보여줘"
3. 연결 실패 시: `/doctor` 명령어로 진단

### 일상 운영

```
☐ "Claude Code의 도구 조작"이 팀 권한 정책에 맞는가?
☐ Slack에 게시된 메시지가 적절한 레벨의 정보인가?
☐ Linear의 자동 생성 티켓이 충분히 상세한가?
```

## 실습 과제

### Level 2: MCP 2개 이상 연결 후 실전 체험

**목표**: 본인 프로젝트의 실제 도구 2개 이상을 연결하고,
PRD 작성부터 티켓 생성까지를 한 번에 실행하기

**진행 방식**:

1. **도구 연결** (10분)

- Notion / Linear 중 최소 2개 선택
- 터미널에 `claude mcp add --transport http <이름> <주소>` 입력
- `/mcp` 로 로그인(OAuth) 인증 완료

2. **Claude Code 테스트** (10분)

- 각 도구에 간단한 명령으로 연결 확인
- 예: "Notion의 [페이지명] 내용을 읽어줄래?"

3. **실전 PRD 작성** (20분)

- 본인 팀의 실제 프로젝트 1개 선택
- Claude Code에서 PRD 작성 요청
- Notion에 자동 게시

4. **티켓 자동 생성** (15분)

- PRD를 기반으로 Linear Epic/Story 자동 생성
- 예상 포인트 추정 포함

5. **결과 검토**

- Notion: PRD가 제대로 게시됐는가?
- Linear: 티켓이 개발팀이 이해할 수 있는 단위로 분해됐는가?
- 소요 시간: 실제로 몇 분 단축됐는가?

**제출 자료** (선택):

- 스크린샷: 생성된 Notion 페이지 + Linear Epic/Story
- 소감문 (1줄): "이전에는 ○○시간 걸렸는데, 이제는 ○○시간만 걸린다"

## 바이브코딩 필수 MCP & 플랫폼 목록

비개발자 PM이 Claude Code로 직접 제품을 만들 때(바이브코딩) 연결하면 좋은 MCP와 플랫폼입니다.
각 MCP를 연결하면 Claude Code가 해당 플랫폼을 자연어로 직접 제어할 수 있습니다.

### 데이터베이스·백엔드

| MCP | 설치 명령어 | 주요 기능 |
|---|---|---|
| **Supabase** | `claude mcp add supabase npx -- -y @supabase/mcp-server-supabase` | DB 쿼리·테이블 생성·RLS 정책·Auth 설정 |
| **PostgreSQL** | `claude mcp add postgres npx -- -y @modelcontextprotocol/server-postgres` | SQL 직접 실행·스키마 조회 |

**Supabase 활용 예시:**

```
"현재 users 테이블에 last_login 컬럼을 추가하고, 오늘 접속한 유저 수를 조회해줘"
```

### 배포·호스팅

| MCP | 설치 명령어 | 주요 기능 |
|---|---|---|
| **Vercel** | `claude mcp add vercel npx -- -y @vercel/mcp` | 프로젝트 배포·도메인 설정·환경변수 관리 |
| **Netlify** | `claude mcp add netlify npx -- -y @netlify/mcp` | 사이트 배포·빌드 설정·폼 연동 |
| **Cloudflare** | `claude mcp add cloudflare npx -- -y @cloudflare/mcp-server-cloudflare` | Workers·Pages 배포·DNS 관리·R2 스토리지 |

### 서버·인프라

| MCP/플랫폼 | 특징 | 바이브코딩 활용 |
|---|---|---|
| **Railway** | 컨테이너 기반, PostgreSQL 포함 | `claude mcp add railway` — 서비스 배포·DB 생성 |
| **Render** | 무료 티어 있는 호스팅 | 서버 배포·환경변수 설정 |

### 개발 생산성

| MCP | 설치 명령어 | 주요 기능 |
|---|---|---|
| **context7** | `claude mcp add context7 -- npx -y @upstash/context7-mcp` | 최신 공식 문서를 Claude에 실시간 주입 |
| **GitHub** | `claude mcp add github npx -- -y @modelcontextprotocol/server-github` | 이슈·PR·코드 검색·커밋 |

### Insforge (올인원 설정)

**Insforge**는 Supabase, Vercel, GitHub 등의 복잡한 MCP 연동을 대시보드 한 곳에서 관리할 수 있는 도구입니다.

```bash
# Insforge를 사용하면 Claude Code에서 99% 설정 가능
claude mcp add insforge npx -- -y @insforge/mcp
```

특히 Supabase 세부 설정(Edge Functions, 인증 정책, 트리거)을 Claude Code 안에서 바로 할 수 있어 바이브코딩 시 큰 시간 절약이 됩니다.

### 권장 MCP 설치 순서 (바이브코딩 시작 시)

```bash
# 1단계: 코드 + 문서
claude mcp add context7 -- npx -y @upstash/context7-mcp
claude mcp add github npx -- -y @modelcontextprotocol/server-github

# 2단계: 데이터베이스
claude mcp add supabase npx -- -y @supabase/mcp-server-supabase

# 3단계: 배포
claude mcp add vercel npx -- -y @vercel/mcp
# 또는
claude mcp add cloudflare npx -- -y @cloudflare/mcp-server-cloudflare

# 설치 확인
claude mcp list
```

> **팁**: 모든 MCP를 한 번에 설치할 필요는 없습니다. 프로젝트가 필요로 하는 플랫폼부터 단계적으로 추가하세요.
