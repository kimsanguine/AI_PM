# 3.8 ralph-loop로 반복 리서치 자동화하기

> **핵심 개념**: PM이 하는 반복 리서치(경쟁사 50개 리스트업, 잠재 고객 100곳 검증, 공개 데이터 크롤링)는 구조는 단순하지만 손이 많이 갑니다. ralph-loop는 Claude Code 안에서 "조건이 달성될 때까지 알아서 반복"하는 자율 루프를 만들어 줍니다.

---

## ralph-loop란

**ralph-loop**는 Claude Code의 플러그인 스킬입니다. 일반 슬래시 명령이 한 번 실행되고 끝나는 것과 달리, ralph-loop는 지정한 완료 조건(`--completion-promise`)이 충족될 때까지 Claude가 스스로 반복해 작업합니다.

핵심 플래그 두 가지만 기억하면 됩니다.

| 플래그 | 역할 | 예시 |
| :--- | :--- | :--- |
| `--max-iterations N` | 최대 반복 횟수 상한 (무한 루프 방지) | `--max-iterations 30` |
| `--completion-promise "조건"` | 이 조건이 참이 되면 루프 종료 | `--completion-promise "DONE"` |

---

## 설치 방법

Claude Code 플러그인 마켓플레이스에서 ralph-loop를 추가합니다.

```shell
/plugin marketplace add anthropics/claude-code
/plugin install ralph-loop@ai-pm
/reload-plugins
```

설치 후 `/ralph-loop`를 입력해 사용 가능한 옵션이 표시되면 정상입니다.

---

## 권한 설정

ralph-loop가 웹 검색과 페이지 수집을 수행하려면 `settings.local.json`에 권한을 선언해야 합니다. 이 파일은 개인 설정이므로 git에 커밋하지 않습니다.

```json
{
  "permissions": {
    "allow": [
      "WebFetch(*)",
      "WebSearch(*)"
    ]
  }
}
```

> **settings.local.json이 없다면?** 프로젝트 루트 `.claude/` 폴더에 새로 만들면 됩니다. 팀 공유 설정인 `settings.json`이 아닌 `settings.local.json`에 적어야 개인 권한이 팀 저장소에 노출되지 않습니다.

---

## PM 실전 예시 — "한국 B2B 챗봇 사용 기업 100개 검증"

**목표**: 국내 B2B 챗봇 솔루션을 실제로 도입한 기업 100곳을 CSV로 정리.

직접 하면 검색 → 링크 클릭 → 도입 여부 확인을 수백 번 반복해야 합니다. ralph-loop에 맡기면 아래 과정이 자율 반복됩니다.

**매 반복(iteration) 흐름:**

```
1. companies.csv 현재 상태 파악 (검증 완료 몇 개?)
2. WebSearch로 후보 기업 탐색
   예: "B2B 챗봇 도입 기업 인터뷰 site:zdnet.co.kr OR site:bloter.net"
3. WebFetch로 랜딩페이지 또는 도입 사례 페이지 확인
4. 도입 근거(보도자료·케이스스터디) 확인 통과 시 CSV에 추가
5. 검증 완료 기업이 100개 미만이면 다음 반복 진행
```

**실행 명령:**

```shell
/ralph-loop \
  --max-iterations 30 \
  --completion-promise "companies.csv에 검증 완료 기업이 100개 이상 기록되면 DONE" \
  "한국 B2B 챗봇 솔루션을 도입한 기업을 검색·검증해 companies.csv에 추가해 줘.
   필드: 기업명, 사용 솔루션, 근거 URL, 확인일자.
   반드시 공개된 사례 근거(보도자료, 케이스스터디, 공식 발표)가 있는 기업만 포함."
```

Claude는 이 명령을 받으면 검색 → 수집 → 검증 → 기록 → 상태 확인을 반복하고, 100개가 채워지거나 30회 반복에 도달하면 멈춥니다.

## PM 활용 시나리오

| 시나리오 | 조건 예시 | 산출물 |
| :--- | :--- | :--- |
| 경쟁사 리스트업 | "특정 기능 보유 경쟁사 N개 확인" | `competitors.csv` |
| 잠재 고객 리드 수집 | "타깃 규모·업종 조건 충족 기업 50곳" | `leads.csv` |
| 공개 데이터 반복 크롤링 | "채용공고 100건 정규화 완료" | `jobs.json` |
| 레퍼런스 리서치 | "논문·아티클 30건 요약 완료" | `references.md` |

## 주의사항 — 무한 루프 방지

`--max-iterations`를 반드시 설정하세요. 완료 조건이 달성되지 않을 경우 루프가 계속 돌 수 있습니다. 권장 상한은 작업 복잡도에 따라 아래를 참고합니다.

- 단순 검색 반복: `--max-iterations 20`
- 페이지 수집 포함: `--max-iterations 30`
- 대용량 크롤링: `--max-iterations 50` (토큰 비용 사전 확인 권장)

루프가 중간에 끊기더라도 CSV·JSON에 이미 기록된 데이터는 유지됩니다. 재실행하면 이전 상태를 읽어 이어서 진행합니다.

## 3.5 n8n 자동화와의 차이

ralph-loop와 n8n은 "자동화"라는 말을 공유하지만 용도가 다릅니다.

| 기준 | ralph-loop | n8n ([3.5](3.5-automation-n8n.md)) |
| :--- | :--- | :--- |
| 실행 위치 | Claude Code 안에서 단발성 집중 수집 | 외부 서버, 스케줄 기반 배경 자동화 |
| 트리거 | PM이 명령을 내릴 때 | 시간(cron), 이벤트(웹훅) |
| 적합 작업 | 한 번의 집중 수집·검증·리스트업 | 매일 반복되는 모니터링·알림 |
| 종료 조건 | 조건 달성 또는 최대 반복 도달 | 스케줄이 취소될 때까지 계속 |
| 설정 난도 | Claude Code 명령 한 줄 | 워크플로 노드 구성 필요 |

**선택 기준**: "지금 당장 N건을 채워야 한다" → ralph-loop. "매주 월요일에 새 데이터를 자동으로 갱신해야 한다" → n8n.

---

## 정리

| 질문 | 답 |
| :--- | :--- |
| ralph-loop가 하는 일 | 완료 조건 달성까지 Claude가 자율 반복 |
| 무한 루프 방지 | `--max-iterations N` 상한 필수 |
| 권한 설정 위치 | `.claude/settings.local.json` (WebFetch·WebSearch) |
| n8n과 차이 | ralph-loop = 단발 집중 수집, n8n = 스케줄 배경 자동화 |

> ralph-loop는 [3.4 커스텀 스킬](3.4-custom-skills.md)에서 배운 스킬 구조 위에서 동작합니다. 스킬이 "무엇을 하라"를 정의한다면, ralph-loop는 "달성될 때까지 반복하라"는 제어 레이어를 더합니다.
