# FAQ·트러블슈팅 허브

> **이 문서의 용도**: 비개발자 PM이 가장 자주 막히는 지점을 한곳에 모았습니다. 각 항목은 **증상 → 원인 → 해결** 순서입니다. 먼저 아래 첫 30분 Quickstart로 끝까지 한 번 가본 뒤, 막히는 지점만 골라 읽으세요.

---

## 첫 30분 Quickstart

설치부터 첫 결과물까지 가장 짧은 경로입니다. **터미널이 익숙하지 않다면 트랙 A**, CLI에 편하다면 **트랙 B**를 따라가세요.

### 트랙 A — 데스크톱 앱/IDE (터미널 없이)

- [ ] **1. 앱 또는 확장을 설치한다.** [claude.ai/code](https://claude.ai/code)에서 데스크톱 앱을 다운로드해 설치하거나, VS Code·JetBrains에서 "Claude Code" 확장을 검색해 설치합니다.
- [ ] **2. 프로젝트 폴더를 연다.** 데스크톱 앱에서 폴더를 열거나, IDE에서 기존 프로젝트를 열고 Claude Code 사이드패널을 활성화합니다.
- [ ] **3. 첫 결과물을 만든다.** 사이드패널에 `이 폴더 구조를 파악하고 INDEX.md를 만들어줘`라고 입력하면 첫 결과물이 나옵니다.

### 트랙 B — CLI

- [ ] **1. 설치한다.** macOS·Linux는 `curl -fsSL https://claude.ai/install.sh | bash`, Windows PowerShell은 `irm https://claude.ai/install.ps1 | iex`.
- [ ] **2. 설치를 확인한다.** 터미널에 `claude --version`을 입력해 버전 번호가 나오면 성공.
- [ ] **3. 프로젝트 폴더로 이동한다.** 작업할 폴더에서 시작해야 그 폴더의 파일을 읽고 씁니다.
- [ ] **4. 실행하고 로그인한다.** `claude`를 입력하면 브라우저가 열립니다. Pro·Max·Team·Enterprise 계정으로 로그인하세요. 무료 Claude.ai 플랜으로는 사용할 수 없습니다.
- [ ] **5. 상태를 점검한다.** 시작이 안 되면 셸에서 `claude doctor`, 시작된 뒤라면 `/doctor`로 설치·설정·인증을 한 번에 확인.
- [ ] **6. 첫 결과물을 만든다.** 자료 파일 하나를 `@파일명`으로 붙여 `이 파일을 한 단락으로 요약해줘.`라고 입력하면 첫 결과물이 나옵니다.

> 막히면 아래 해당 항목으로 가세요. 그래도 안 되면 어느 단계든 `/doctor`(또는 `claude doctor`)가 가장 빠른 진단입니다.

## 필수 단축키 & CLI 플래그

### 키보드 단축키

| 단축키 | 기능 | 언제 쓰나 |
|---|---|---|
| `ESC` | 현재 실행 중인 작업 즉시 취소 | Claude가 잘못된 방향으로 가고 있을 때 |
| `ESC` + `ESC` (연속) | 직전 메시지 회귀 | 보낸 메시지를 보내기 직전으로 롤백 |
| `Shift` + `Enter` | 줄바꿈 입력 (`/terminal-setup` 실행 후) | 긴 프롬프트 여러 줄 입력 |
| `Shift` + `Tab` | 모드 전환 | 일반 → 자동승인 → 플래닝 → 스킵 순환 |
| `Tab` | 경로 자동완성 | 파일·폴더 경로 입력 시 |

> **ESC+ESC 활용 팁**: Claude가 잘못된 가정으로 작업을 시작했다면, ESC+ESC로 그 메시지를 보내기 전 상태로 돌아가 다시 입력할 수 있습니다. 잘못된 컨텍스트가 누적되는 것을 방지합니다.

### 경로 탭 자동완성

`@`로 파일을 참조할 때 `Tab`을 누르면 자동완성이 됩니다.

```
@doc  [Tab]  →  @docs/part2-basics/2.2-modes-and-depth.md
```

실무에서 유용한 경우:

- CLAUDE.md 안에서 `@파일명`으로 다른 파일 참조
- 긴 경로를 정확하게 입력할 때
- 파일 이름이 기억나지 않을 때 일부만 입력 후 Tab

### CLI 실행 플래그

| 플래그 | 명령어 | 설명 |
|---|---|---|
| `-c` / `--continue` | `claude -c` | **직전 세션** 바로 재진입 (현재 디렉토리 기준 가장 최근) |
| `-r` / `--resume` | `claude -r "세션명"` | **특정 세션** 선택해서 재진입 (전체 세션 중 선택) |

```bash
# 어제 하던 작업 바로 이어서
claude -c

# 여러 세션 중 habix legal 프로젝트 세션 재개
claude -r "habix legal"
```

**-c와 -r의 차이:**

- `-c`: 현재 디렉토리 기준으로 가장 최근 세션만 잡음
- `-r`: 전체 세션 중에서 선택 (다른 프로젝트의 세션도 접근 가능)

## Explanatory Output: 설명형 출력 포맷 설정

Claude Code의 출력 스타일은 `/config`에서 설정할 수 있습니다. **Explanatory(설명형)**은 초급자에게 특히 중요한 설정입니다.

### 설정 방법

```
/config
→ Output Style: Explanatory
```

### Explanatory 스타일이 하는 것

일반 출력과 비교:

| 일반 출력 | Explanatory 출력 |
|---|---|
| 코드만 제공 | 코드 + 각 줄 설명 |
| 결과만 제공 | 결과 + "왜 이렇게 했는지" 설명 |
| 수정만 함 | 수정 전후 차이 설명 포함 |

### 초급자에게 왜 중요한가

Claude Code를 처음 쓸 때 가장 어려운 것은 **"Claude가 뭘 하는지 모르는 상태"**입니다.
Explanatory 모드는 Claude가 작업마다 "왜, 어떻게"를 설명해 주어, 학습하면서 작업을 진행할 수 있습니다.

```
# 예시: 파일 수정 요청 후 Explanatory 출력

"auth.ts에서 JWT 검증 로직을 추가해줘"

Claude 출력:
> ★ Insight: JWT 검증은 미들웨어로 분리하는 것이 관심사 분리 원칙에 맞습니다.
> 기존 route handler에 직접 넣으면 중복 코드가 생깁니다.
>
> [코드 수정]
>
> 변경 요약: verifyToken 미들웨어 추가, 3개 라우트에 적용
```

### 다른 설정 옵션

`/config`에서 함께 설정하면 좋은 것들:

| 설정 | 권장값 | 이유 |
|---|---|---|
| Output Style | Explanatory | 학습 + 작업 동시에 |
| Language | 한국어 | UI 텍스트·설명을 한국어로 |
| Auto-connect to IDE | true | 현재 작업 파일 컨텍스트 자동 반영 |
| Enable remote control | true | IDE에서 Claude Code 제어 |

## 설치가 실패할 때

**증상**: 설치 명령을 실행했는데 에러가 나거나, 끝났는데 `claude --version`이 동작하지 않는다.

**원인**: 운영체제·셸에 맞지 않는 명령을 복사했거나, 설치는 됐지만 설치 폴더가 PATH(명령을 찾는 경로 목록)에 없는 경우가 가장 많습니다.

**해결**:

- Windows에서 `irm ... not recognized`가 나오면 CMD에 있는 것입니다. PowerShell을 열고 `irm https://claude.ai/install.ps1 | iex`를 실행하세요. 반대로 `&&` 관련 에러가 나오면 PowerShell에서 CMD용 명령을 쓴 것입니다.
- 설치는 됐는데 `command not found: claude`가 나오면 PATH 문제입니다. 가장 간단한 해결은 **터미널을 닫고 다시 여는 것**입니다.
- 그래도 안 되면 셸에서 `claude doctor`를 실행해 진단 결과를 확인하세요.
- 대안 설치도 가능합니다. macOS는 Homebrew(`brew install --cask claude-code`), Windows는 WinGet(`winget install Anthropic.ClaudeCode`).

자세한 단계는 본문 [1.3 설치부터 첫 대화까지](../part1-foundations/1.3-install-and-first-run.md)를 참고하세요.

## 권한 프롬프트가 계속 뜰 때

**증상**: 같은 종류의 작업을 할 때마다 "이 작업을 허용할까요?" 확인 창이 반복해서 뜬다.

**원인**: Claude Code는 기본적으로 변경·실행 작업마다 사용자 승인을 받습니다(검토 우선 설계). 매번 뜨는 것은 정상 동작이지만, 신뢰하는 작업이 반복되면 번거로울 수 있습니다.

**해결**:

- 세션 안에서 `/permissions`를 입력하면 현재 허용 규칙을 보고 수정할 수 있습니다.
- 특정 작업을 매번 자동 허용하려면 `settings.json`의 `permissions.allow` 목록에 규칙을 추가합니다. 규칙은 `도구` 또는 `도구(범위)` 형태입니다.

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run test:*)",
      "Read(./src/**)"
    ]
  }
}
```

- 허용 범위는 좁게 잡는 것이 안전합니다. 모든 작업을 무조건 통과시키는 방식은 데이터 변경·삭제 위험이 있으므로, 검토가 필요한 작업(PRD 수정, 고객 대면 콘텐츠 등)에는 권장하지 않습니다.

루프 안에 언제 머물러야 하는지는 본문 [2.6 휴먼 인 더 루프](../part2-basics/2.6-human-in-the-loop.md)를 참고하세요.

## MCP 연결이 안 될 때

**증상**: `claude mcp add`로 도구를 추가했는데 연결이 안 되거나, `/mcp`에 서버가 보이지 않는다.

**원인**: 설정이 적용되지 않았거나, 인증 토큰·주소가 틀렸거나, 서버가 시작 시 로드되지 않은 경우입니다.

**해결**:

- 세션 안에서 `/mcp`를 입력해 연결된 서버와 상태를 먼저 확인하세요.
- `/doctor`는 설치·설정과 함께 MCP 구성도 한 번에 점검합니다.
- 추가가 안 되면 `claude mcp add` 명령의 도구 이름·주소를 다시 확인하세요. JSON을 직접 편집할 필요 없이 명령 한 줄로 등록됩니다.

연결 절차와 PM이 자주 쓰는 4가지 MCP는 본문 [3.1 MCP 연동](../part3-advanced/3.1-mcp-integration.md)을 참고하세요.

## 인증·로그인이 안 될 때

**증상**: 로그인이 루프에 빠지거나, `OAuth error`, `403 Forbidden`이 뜨거나, 세션마다 다시 로그인하라고 한다.

**원인**: 토큰 만료, 브라우저가 로컬 콜백 서버에 닿지 못하는 환경(SSH·WSL·컨테이너), 또는 구독이 비활성인 경우입니다.

**해결**:

- 대부분은 깨끗하게 다시 인증하면 해결됩니다. `/logout`으로 로그아웃 → Claude Code 종료 → `claude`로 재시작 후 다시 로그인.
- 토큰 만료로 다시 로그인하라고 하면 `/login`을 실행하세요. 자주 반복되면 시스템 시계가 정확한지 확인하세요(토큰 검증이 시각에 의존).
- 브라우저가 자동으로 열리지 않으면 로그인 화면에서 `c`를 눌러 URL을 복사한 뒤 직접 브라우저에 붙여넣으세요.
- `403 Forbidden`이 뜨면 구독이 활성 상태인지 [claude.ai/settings](https://claude.ai/settings)에서 확인하세요. 무료 Claude.ai 플랜으로는 Claude Code를 쓸 수 없습니다.

## API 키 문제일 때

**증상**: 구독이 멀쩡한데 `This organization has been disabled` 같은 에러가 나거나, 의도한 인증 방식과 다르게 동작한다.

**원인**: 이전 직장·프로젝트에서 셸 프로필에 남아 있는 `ANTHROPIC_API_KEY` 환경 변수가 구독 로그인보다 우선 적용되는 경우가 흔합니다.

**해결**:

- 어떤 인증 방식이 활성인지 `/status`로 먼저 확인하세요.
- 구독으로 쓰고 싶다면 환경 변수를 해제하고 다시 실행합니다.

```bash
unset ANTHROPIC_API_KEY
claude
```

- 영구적으로 지우려면 `~/.zshrc`, `~/.bashrc`, `~/.profile`에서 `export ANTHROPIC_API_KEY=...` 줄을 제거하세요(Windows는 PowerShell 프로필 `$PROFILE`과 사용자 환경 변수).
- 직접 API 키로 쓰는 경우, 키는 [platform.claude.com](https://platform.claude.com)에서 발급합니다.

## 응답이 이상할 때

**증상**: 답이 엉뚱하거나, 대화가 길어질수록 맥락을 놓치거나, 같은 곳을 빙빙 돈다.

**원인**: 대화가 길어져 컨텍스트(한 번에 기억하는 분량)가 가득 찼거나, 큰 파일 하나가 컨텍스트를 차지한 경우입니다.

**해결**:

- 대화가 길어졌다면 `/compact`로 맥락을 압축하세요. 무엇을 남길지 지정할 수도 있습니다(예: `/compact 계획과 결정사항만 남겨줘`).
- 주제가 완전히 바뀌면 `/clear`로 대화를 비우고 새로 시작하세요.
- 큰 파일 때문이라면 파일 전체 대신 **필요한 부분만**(특정 구간·항목) 읽게 하거나, 그 작업을 서브에이전트에게 맡겨 별도 컨텍스트에서 처리하게 하세요.
- 멈춘 것처럼 보이면 Ctrl+C로 현재 작업을 취소하세요. 대화는 사라지지 않으며, 같은 폴더에서 `claude --resume`으로 이어갈 수 있습니다.

깊이·모드 조절은 본문 [2.2 모드와 사고 깊이](../part2-basics/2.2-modes-and-depth.md)를 참고하세요.

## 한도에 도달했을 때

**증상**: 사용량 한도에 도달했다는 안내가 뜨고 더 진행되지 않는다.

**원인**: 플랜별로 사용량 한도가 있습니다. 긴 작업을 연속으로 돌리면 한도에 먼저 닿을 수 있습니다.

**해결**:

- 현재 어떤 인증·플랜으로 쓰고 있는지 `/status`로 확인하세요.
- 한도는 일정 시간이 지나면 초기화됩니다. 급하지 않은 작업은 잠시 뒤에 다시 시도하세요.
- 같은 결과를 더 적은 호출로 얻으려면, 한 번에 다 시키지 말고 단계를 나누고 불필요하게 큰 파일을 통째로 읽히지 마세요(`/compact` 병행).
- 매일 무겁고 긴 작업을 돌리게 됐다면 상위 플랜을 검토할 시점입니다. 최신 플랜·가격은 [claude.com/pricing](https://claude.com/pricing)에서 확인하세요.

## 외부 스킬·플러그인을 받아 설치해도 안전한가요?

외부에서 스킬이나 플러그인을 받아 설치하기 전에 다음 4단계를 확인하세요.

1. `SKILL.md`(또는 `CLAUDE.md`) 파일을 직접 열어 어떤 명령어·권한을 요청하는지 확인합니다.
2. `scripts/` 폴더의 번들 스크립트에 외부 네트워크 호출이나 파일 삭제 로직이 있는지 확인합니다.
3. 제작자의 GitHub 프로필과 커뮤니티 평판을 확인합니다.
4. 처음엔 `--permission-mode ask` 모드에서 테스트합니다.

> 보안 연구(ToxicSkills)에 따르면 공개 스킬의 상당수에서 prompt injection 취약점이 발견된 바 있습니다. 신뢰할 수 없는 출처의 스킬은 주의가 필요합니다.

---

## 스킬을 설치했는데 자동으로 실행이 안 됩니다

다음 순서로 확인하세요.

1. `/skill-name` 형식으로 직접 입력해 수동 실행을 먼저 시도합니다.
2. `/help`로 스킬이 등록되어 있는지 확인합니다.
3. `SKILL.md`의 `description` 필드가 너무 모호하지 않은지 확인합니다. Claude가 언제 쓸지 판단하지 못할 수 있습니다.
4. `CLAUDE.md`에 "X 상황이면 반드시 /skill-name 사용"이라고 명시해 트리거 조건을 구체화합니다.
5. 그래도 안 되면 슬래시 명령어 수동 호출을 기본 습관으로 삼으세요.

## MCP 서버 설치 후 제대로 연결됐는지 어떻게 확인하나요?

- `claude mcp list`로 등록된 서버 목록을 먼저 확인합니다.
- 실제 동작 테스트: "Notion의 최근 페이지 1개 제목을 가져와봐"처럼 데이터를 직접 요청해 응답이 오는지 봅니다.
- 연결 실패 시: `/doctor` 명령어로 MCP 구성 포함 전체 진단을 실행합니다.

연결 절차와 PM이 자주 쓰는 MCP는 본문 [3.1 MCP 연동](../part3-advanced/3.1-mcp-integration.md)을 참고하세요.

## 토큰 비용이 걱정됩니다. 어떻게 관리하나요?

- 하단 상태줄에서 현재 컨텍스트 크기를 실시간으로 확인할 수 있습니다.
- 긴 작업 후 `/compact`로 컨텍스트를 압축하면 불필요한 토큰 소모를 줄일 수 있습니다.
- 주제가 바뀔 때 `/clear`로 초기화하면 컨텍스트가 불필요하게 누적되지 않습니다.
- 모델별 비용 차이가 크므로, 탐색·초안 작업은 `/model sonnet` 전환을 고려하세요.
