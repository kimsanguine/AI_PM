# 설치부터 첫 대화까지

## 사전 요구사항

Claude Code를 시작하기 전에 다음 두 가지가 필요하다.

### 1. 유료 계정 (플랜 선택)

Claude Code는 무료 Claude.ai 플랜으로는 사용할 수 없다. 다음 중 하나가 필요하다.

- **Claude Pro** — 개인용 입문 플랜. 가볍게 시작하기 좋다.
- **Claude Max** — 사용량 한도가 더 큰 개인용 플랜. 매일 길게 쓴다면.
- **Team** — 팀 단위 구독.
- **Enterprise** — 조직 단위 관리·보안 기능.
- **Console (API)** — [Anthropic Console](https://console.anthropic.com/) 계정으로 API 사용량 기준 과금.

타사 모델 제공자([Amazon Bedrock](https://platform.claude.com/docs/en/build-with-claude/claude-in-amazon-bedrock), [Google Vertex AI](https://platform.claude.com/docs/en/build-with-claude/claude-on-vertex-ai))를 통해 쓸 수도 있다.

**처음이라면 작게 시작하세요.** 개인 PM이 혼자 써본다면 Pro로 충분히 감을 잡을 수 있고, 매일 길고 무거운 작업을 돌리게 되면 그때 Max로 올리면 됩니다. 가격은 변동될 수 있으니 [claude.com/pricing](https://claude.com/pricing)에서 최신 금액을 확인하세요.

### 2. 터미널 접근 (CLI로 쓸 경우)

CLI로 쓸 거라면 터미널이 필요하다. macOS · Linux는 기본 터미널, Windows는 PowerShell · CMD · Windows Terminal을 쓴다.

> 검은 화면이 부담된다면, 1.2의 "두 가지 입구"에서 본 것처럼 **데스크톱 앱이나 VS Code · Cursor 확장**으로 시작할 수 있습니다. 그 경우 이 챕터의 CLI 설치 단계를 건너뛰어도 됩니다.
>
> **Node.js는 필수가 아니다.** 아래 권장 방식(native installer)은 Node.js 없이 단독 실행 바이너리를 설치한다. Node.js 18 이상은 `npm`으로 설치하는 경로([고급 설치](#고급-설치-옵션) 참고)에서만 필요하다.

---

## 설치 방법 A — 터미널 없이 시작하기

1. [claude.ai/code](https://claude.ai/code) 에서 데스크톱 앱 다운로드
2. 또는 VS Code / JetBrains에서 "Claude Code" 확장 검색 후 설치
3. 프로젝트 폴더를 열면 바로 사용 가능

> 터미널이 익숙하지 않다면 이 방법을 먼저 사용하세요.

---

## 설치 방법 B — CLI(터미널) 설치

가장 권장되는 방식은 **native installer**다. Node.js가 필요 없고, 자동으로 백그라운드 업데이트된다.

### macOS · Linux · WSL (권장)

터미널을 열고:

```
curl -fsSL https://claude.ai/install.sh | bash
```

이 명령어는:

1. Claude Code 단독 실행 바이너리를 다운로드
2. `~/.local/bin`에 설치
3. 이후 자동으로 최신 버전 유지

### Windows PowerShell

PowerShell을 열고 (관리자 권한 불필요):

```
irm https://claude.ai/install.ps1 | iex
```

WSL2를 쓰고 있다면 WSL 터미널 안에서 macOS · Linux와 같은 명령을 쓴다.

### 다른 설치 경로 (한 줄 요약)

선호하는 패키지 매니저가 있다면 아래 중 하나를 써도 된다.

- **Homebrew (macOS)**: `brew install --cask claude-code`
- **WinGet (Windows)**: `winget install Anthropic.ClaudeCode`
- **apt (Debian · Ubuntu)**: 서명된 저장소 등록 후 `sudo apt install claude-code`
- **dnf (Fedora · RHEL)**: 저장소 등록 후 `sudo dnf install claude-code`

> Homebrew · WinGet · apt · dnf 설치본은 자동 업데이트되지 않으므로 직접 업그레이드해야 한다. 저장소 등록 절차 등 자세한 내용은 [code.claude.com/docs/en/setup](https://code.claude.com/docs/en/setup)을 참고하라.

### 설치 완료 확인

어느 플랫폼이든, 설치 후:

```
$ claude --version
```

버전 번호가 나오면 설치 완료다.

### 고급 설치 옵션

**npm 글로벌 패키지 (Node.js 18+ 필요)**

이미 Node.js 18 이상이 깔려 있고 npm 방식을 선호한다면:

```
npm install -g @anthropic-ai/claude-code
```

이 경로에서만 Node.js가 필요하다. 권장 방식(native installer)은 Node.js 없이 동작한다. `sudo npm install -g`는 권한·보안 문제를 일으킬 수 있으니 쓰지 않는다.

## 초기 설정

처음 Claude Code를 실행하면 **로그인**을 요청받는다.

### 1단계: 첫 실행

아무 프로젝트 폴더에서나:

```
$ claude
```

### 2단계: 인증 (기본은 브라우저 로그인)

`claude`를 실행하면 안내에 따라 **브라우저가 열리고, Pro · Max 등 구독 계정으로 로그인**한다. 별도의 API 키를 발급해 붙여넣을 필요가 없다 — 구독 계정으로 로그인하는 것이 기본 경로다.

> 🖼️ 스크린샷 자리: 브라우저 로그인 동의 화면 → 터미널로 자동 복귀

**API 키 경로 (옵션)**: Anthropic Console(API) 계정을 쓰거나, Bedrock · Vertex 같은 타사 제공자를 통해 쓸 때만 API 키 방식이 필요하다. 이 경우 [console.anthropic.com](https://console.anthropic.com/)에서 키를 발급해 환경 변수 등으로 설정한다. 계정 유형별 인증 방법은 [code.claude.com/docs/en/setup](https://code.claude.com/docs/en/setup)을 참고하라.

### 3단계: 상태 확인

로그인한 후, 설치·설정을 점검하려면:

```
$ claude doctor

✓ Authentication valid
✓ Network connection OK
✓ Claude Code ready to use

Model: claude-opus-4-8
```

모든 항목이 ✓라면 설정 완료. (명령은 슬래시 없이 `claude doctor`다.)

## 첫 대화 시작하기

### 프로젝트 폴더로 이동

Claude Code는 현재 폴더를 기준으로 파일을 읽고 쓴다. 따라서 프로젝트 폴더에서 시작하는 게 좋다:

```
$ cd ~/my-product  # 자신의 프로젝트 폴더
$ claude
```

### Claude와의 첫 대화

이제 프롬프트가 나타난다:

```
╭──────────────────────────────────────────────╮
│ ✻ Welcome to Claude Code!                   │
│   Type /help for available commands          │
│   /exit to quit                               │
╰──────────────────────────────────────────────╯

Project: my-product
Location: /Users/yourname/my-product

>
```

첫 질문을 해보자. 예를 들어, 프로젝트의 README가 있다면:

```
> @README.md 이 프로젝트를 3줄로 요약해줘
```

Claude가 답한다:

```
README.md를 읽었습니다.

당신의 프로젝트는:

1. 고객 피드백을 관리하는 SaaS 플랫폼으로,
   여러 채널(이메일, 슬랙, 웹폼)의 피드백을
   한곳에 모아 분석하는 도구입니다.

2. 주요 사용자는 스타트업의 제품 매니저들이고,
   각 피드백에 대해 자동으로 감정 분석과
   분류를 제공합니다.

3. 현재 알파 단계이며, 초대 사용자 50명 정도가
   테스트 중입니다.

혹시 더 알고 싶은 점이 있으신가요?

>
```

성공이다. Claude와 대화할 준비가 끝났다.

## 실습 1: 간단한 파일 요약

이제 조금 더 복잡한 작업을 해보자.

### 준비

프로젝트 폴더에 다음과 같은 파일들이 있다고 가정:

- `meeting-notes.md`: 어제 회의록
- `user-feedback.csv`: 수집한 유저 피드백 데이터

### 명령

```
> @meeting-notes.md @user-feedback.csv
  회의 내용과 유저 피드백을 함께 분석해서,
  우리가 해야 할 일을 5개 항목으로 정리해줘
```

Claude가 작업한다:

```
두 파일을 분석했습니다.

회의에서 나온 내용:
- Q1 로드맵 확정 필요
- 경쟁사 분석 리포트 필요
- 팀 구성 검토

유저 피드백의 주요 테마:
- 속도 개선 요청 (45%)
- UI 개선 요청 (30%)
- 통합 기능 요청 (25%)

종합하면, 우리가 해야 할 일:

1. [높음] 앱 속도 최적화 - 유저 피드백 1순위
2. [높음] 경쟁사 분석 리포트 완성 - 회의에서 결정
3. [중간] UI/UX 개선 - 피드백 2순위
4. [중간] Q1 로드맵 공식화 - 회의 액션아이템
5. [낮음] 새로운 통합 기능 조사 - 향후 계획

우선순위에 따라 착수 순서를 정하시겠어요?

>
```

### 결과 저장

원하면 결과를 파일로 저장할 수도 있다:

```
> 이 분석을 action-plan.md 파일로 저장해줘

Claude: action-plan.md 파일을 생성했습니다.
파일의 내용은:
# Q1 액션 플랜

## 우선순위 순

### 1. [높음] 앱 속도 최적화
...

파일을 확인하려면: cat action-plan.md

>
```

## 대화 종료 방법

대화를 끝내고 싶을 때:

### 1. `/exit` 명령

```
> /exit

✓ 대화를 종료합니다.
프로젝트 메모가 저장되었습니다 (CLAUDE.md).

$
```

### 2. Ctrl+C 키

```
> ^C

✓ 대화를 종료합니다.

$
```

둘 다 같은 효과다. 이전 대화는 `CLAUDE.md`에 저장되어, 다음에 같은 폴더에서 `claude`를 실행할 때 참고된다.

## 자주 나는 문제 & 해결법

### 문제 1: "claude: command not found"

**원인**: PATH에 등록되지 않았다.

**해결**:

- macOS/Linux:

```
  # 수동으로 PATH 추가
  echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
  source ~/.bashrc
```

- Windows PowerShell: PowerShell을 다시 열어보자.

설치 후 터미널을 재시작하는 게 가장 간단하다.

### 문제 2: 로그인 / 인증 오류

**원인**: 로그인 세션이 만료되었거나, 구독·계정 상태에 문제가 있다.

**해결**:

1. `claude` 명령을 다시 실행해 브라우저 로그인을 재시도한다.
2. 사용 중인 구독(Pro · Max · Team 등)이 활성 상태인지 확인한다.
3. API 키 경로를 쓰는 경우([console.anthropic.com](https://console.anthropic.com/)), 키가 유효한지 확인한다.

### 문제 3: npm 설치본에서 바이너리를 찾지 못함

**원인**: `npm`으로 설치한 경우, 플랫폼별 바이너리가 제대로 연결되지 않았다. (native installer를 썼다면 이 문제는 없다.)

**해결**:

1. 가능하면 권장 방식인 native installer로 다시 설치한다.
2. npm 경로를 유지한다면 `npm install -g @anthropic-ai/claude-code@latest`로 재설치한다.
3. 자세한 진단은 `claude doctor` 출력과 [설치 문제 해결 문서](https://code.claude.com/docs/en/setup)를 참고한다.

### 문제 4: "Network connection failed"

**원인**: 인터넷 연결 문제 또는 Claude API 서버 장애.

**해결**:

1. 인터넷 연결 확인
2. VPN을 사용 중이면, 비활성화 후 시도
3. 몇 분 뒤 다시 시도
